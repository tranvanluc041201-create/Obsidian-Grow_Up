# 数学学习流

> [!note]
> 这是 `10_数学` 的唯一真源入口。
> 日常速查看 `[[10_数学/快速开始]]`，模板入口看 `[[90_导师模式/Math/03_L3_TEMPLATES|数学模板索引]]`。

## 适用范围

- 数学网课学习
- 题目分析
- 证明题
- 错题复盘
- 视频笔记整理

## 触发条件

- 用户明确说“数学视频学习模式”
- 直接发数学课程视频链接 / 本地视频文件
- 要做以下四件事之一：
  - 流程验证
  - 内容整理
  - 诊断提问
  - 笔记输出

## 默认原则

1. **先分析，再整理**
2. **先推导，再结论**
3. **AI 负责提取与压缩，你负责回忆与判断**
4. **默认轻流，不默认重笔记**
5. **不做大而全摘要，优先做复习工具**
6. **卡 5 分钟先标卡点，卡 30 分钟转错题/周复盘**

## 数学视频学习的真源分工

> [!note]
> `数学学习流.md` 负责长期规则与原则。
> `[[99_Templates/数学/视频学习流程模板|视频学习流程模板]]` 只负责执行时的分流卡。

不要把两边都写成完整操作手册。

## 各阶段的最低输入

### 流程验证

- 视频链接 / 本地视频文件

### 内容整理

- 视频链接 / 本地视频文件
- 章节或主题（不知道可以暂空）

### 诊断提问

- 已完成内容整理

### 笔记输出

- 已完成诊断提问

## 人机协作点

1. AI 第一句先给：**当前判定模式 + 判断依据**
2. AI 第二句再给：**已有输入 / 缺失输入 / 推荐下一步**
3. 你先做基于内容的回忆、判断与选择，再让我诊断
4. 我先推荐最该记录的薄弱点和笔记侧重点，你确认
5. 默认先给 `薄弱点诊断 + 笔记骨架`，不是直接正式入库
6. 只有你确认骨架无误后，才进入正式笔记

## 默认工作流

### 0. 先判轻流还是重流

默认走 **轻流**：

- 目标：先学会、先会回忆、先会做题
- 默认产物：`D 版整理 + 诊断/小测 + 薄弱点判断`
- 默认不直接正式记笔记

只有满足以下条件，才升级到 **重流**：

- 你明确要正式笔记或入库
- `5 问诊断` 或小测没过
- 板书特别重，转写不足以支撑理解
- 图形 / 分类讨论 / 连续推导很多
- 第二天回忆失败，或同类题连续错

### 1. 流程验证

- 只验证：下载 / 转写 / 产物生成
- 不问薄弱点
- 不问笔记侧重点
- 不进入 `学习整理助手`

### 2. 内容整理

默认输出 **D 版整理**：

- 内容骨架
- 关键概念 / 公式
- 关键推导 / 例题
- 易错候选
- 关键帧证据

这一步先把内容压缩清楚，不急着定笔记重点。

> [!important]
> `D 版整理` 是 AI 的中间产物，不是默认要保存的正式笔记。
> 它的作用是支撑后续回忆、诊断、小测与是否升级到重流的判断。

### 3. 诊断提问

固定用 **5 问诊断法**：

1. 这节课核心在解决什么
2. 哪个判断最容易混淆
3. 为什么这里能从 A 推到 B
4. 如果现在出题，最可能错在哪一类
5. 这篇笔记最该优先保留哪两类内容

默认形式：

- `选择 / 排序 + 一句理由`

默认薄弱点维度：

- 概念判断
- 推导衔接
- 方法选择
- 条件检查
- 题目迁移

如果用户对第 4 问拿不准，优先补一轮 **2~5 题小测**，不要逼用户空口判断。

### 3.5 轻流默认交付

- `D 版整理`
- `5 问诊断` 或 `2~5 题小测`
- `薄弱点初判`
- `是否需要重流` 的判断

这一步完成后，如果掌握稳定，可以停在这里，不必强行入库。

### 4. 笔记输出

默认先交付：

- 薄弱点诊断
- 推荐笔记侧重点
- 笔记骨架

默认不是直接完整笔记。

### 5. 正式入库（可选）

只有当：

1. 诊断已经完成
2. 已确认这节课确实需要重流沉淀
3. 你确认骨架没问题
4. 你明确要落到 Obsidian

才进入正式笔记。

### 6. 再决定是否加关键帧

默认顺序：

1. 页面元信息 / 讲义 / 字幕
2. 补完整音频转写
3. **内容整理阶段默认开启关键帧**
4. 关键帧默认先走 API `zai-org/GLM-4.6V`
5. 默认把下载上限提到 `1080p`；源站不给时自动退到可下载最高画质
6. 板书特别密时，可加黑板裁剪
7. OCR/板书抽取不稳时，可切 `PaddlePaddle/PaddleOCR-VL-1.5`
8. 需要离线或 API 不可用时，再切本地 WSL `MiniCPM-V-4.5`

#### 不开关键帧

- 只在 `流程验证` 阶段不开
- 只做技术链路检查时不开

#### 开关键帧（默认）

- `内容整理` 默认开启
- 优先保留：
  - 板书定义
  - 辨析例子
  - 公式链
  - 验证过程

#### 加强定向关键帧

- 公式连续变形多
- 黑板板书承载关键信息特别重
- 题目分情况讨论明显
- 图形、曲线、表格不是一句话能说清

> [!warning]
> **默认先用 API `zai-org/GLM-4.6V` 做关键帧分析。**
> 板书 OCR 特别重时，优先补 `PaddlePaddle/PaddleOCR-VL-1.5`；API 不可用时再退回本地 VLM。

### 7. 24 小时复习

第二天只做 10 分钟：

- 遮住笔记，复述定义 / 条件 / 方法
- 回答“为什么这里能这么做”
- 补 1 道小题或 1 段关键推导

### 8. 周复盘

每周至少做 3 件事：

1. 汇总本周错题
2. 统计高频错误模式
3. 选 1 个薄弱专题回炉

## 验收标准

### 流程验证

- [ ] 能下载或读取源视频 / 音频
- [ ] 能成功生成 `metadata.json`
- [ ] 能成功生成 `transcript.json`
- [ ] 能成功生成 `transcript.txt`
- [ ] 已明确这次只是“链路验证”，不是正式学习

### 内容整理

- [ ] 已拿到基础材料
- [ ] 已产出 D 版整理
- [ ] 已补关键帧证据
- [ ] 没有误入 `学习整理助手`
- [ ] 没把 D 版整理误当正式笔记

### 诊断提问

- [ ] 问题确实基于内容整理结果
- [ ] 已能初判薄弱点属于哪一类
- [ ] 已推荐笔记侧重点，而不是让用户空口泛答
- [ ] 用户答不出时，优先补小测而不是硬问

### 笔记输出

- [ ] 默认先给诊断 + 骨架
- [ ] 还没有跳成正式 Obsidian 笔记
- [ ] 只有在用户确认后才进入正式入库

## 止损点

- 只是测链路时，不做完整笔记
- 内容还没整理完时，不提前问笔记侧重点
- 还没做诊断提问时，不直接出正式笔记
- 轻流能解决时，不升级成重流
- 内容整理阶段不要省掉关键帧
- 默认先用 API `zai-org/GLM-4.6V`
- OCR/板书抽取不稳时，先切 `PaddlePaddle/PaddleOCR-VL-1.5`
- API 不可用时，再切本地 WSL `MiniCPM-V-4.5`
- 学科不明确或材料并非数学内容时，转回 `学习中枢`
- 用户只要复习工具时，不要膨胀成大而全课程总结
