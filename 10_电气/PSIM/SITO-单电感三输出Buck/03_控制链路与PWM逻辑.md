# SITO 单电感三输出 Buck｜控制链路与 PWM 逻辑

---

## 一、这条控制链到底长什么样

对每一路输出，控制链都可以抽象成：

```text
目标电压 Vref
→ 与反馈电压 Vfb 做差，得到误差 e
→ PI 控制器输出控制量
→ Limiter 限幅
→ 与 ramp 比较
→ 得到 PWM
→ 控制开关导通时序
```

写成一条更紧凑的链就是：

`Vref - Vfb → e → PI → Limiter → ramp 比较 → PWM`

---

## 二、为什么 PI 输出不能直接等于开关信号

PI 输出本质上不是“开 / 关”命令，而是一个**连续控制量**。

你可以把它理解成：

- `0.2` 表示“我希望占空比大概 20%”
- `0.8` 表示“我希望占空比大概 80%”

也就是说，PI 输出表达的是“力度”，不是最终的数字开关。

真正把这个连续控制量变成开关信号的，是：

- `ramp`
- `比较器`

---

## 三、PWM 比较器到底在做什么

比较器本质是在回答一个问题：

> 当前时刻，控制量 `d` 是否大于载波 `ramp(t)`？

如果把控制量写成 `d`，那么一个常见的理想逻辑是：

```text
当 ramp < d 时，PWM = 1
当 ramp > d 时，PWM = 0
```

所以：

- `d` 越大，高电平持续时间越长
- `d` 越小，高电平持续时间越短

这就是“控制量 → 占空比”的映射。

---

## 四、为什么 ramp 和 Limiter 必须同量纲

这是你这次实验中最关键的知识点之一。

如果：

- `ramp` 是 `0 ~ 1`
- 但 `Limiter` 输出却是 `0 ~ 5V`

那比较器就会失真：

- 要么长期全开
- 要么长期全关
- 要么表现得像 PWM 逻辑反了

所以必须保证比较链条上的量纲一致：

| 环节 | 推荐范围 |
|------|----------|
| PI 输出 | 经过限幅后映射到统一范围 |
| Limiter 输出 | `0 ~ 1` |
| ramp | `0 ~ 1` |
| 最终占空比指令 | `0 ~ 1` |

---

## 五、Limiter 的作用不是“可有可无”

Limiter 至少有两个作用：

### 1. 保证比较器输入在合理范围

如果不做限幅，PI 输出可能超出比较器期望范围，造成 PWM 失真。

### 2. 抑制积分项带来的失控

在大误差下，积分项可能持续累积，导致：

- 输出长期贴边
- 控制器恢复很慢
- 启动时更容易超调

这也是你后面专门整理出 `E3-限幅-运放饱和` 的原因。

---

## 六、从系统角度看三路闭环

你这不是单输出 Buck，而是**单电感三输出**。

这意味着能量不是同时、独立地分给三路，而是要靠时序进行分配。

所以即使每一路都有自己的 PI，真正决定效果的仍然是：

1. 每一路的反馈是否正确
2. 每一路的控制量是否合理
3. PWM 时序是否正确
4. 多路之间的耦合是否可接受

这也是为什么：

- 某一路启动时可能先被“喂饱”
- 另一条路可能短时下探

这不是简单的“PI 参数不好”，而是结构本身就存在耦合。

---

## 七、你这次实验里最值得记住的判断句

### 判断句 1

**PI 输出不是开关信号，而是占空比指令。**

### 判断句 2

**PWM 不是控制器直接给出来的，而是控制量和载波比较后的结果。**

### 判断句 3

**只要 Limiter 和 ramp 不同量纲，比较器就会表现异常。**

### 判断句 4

**单电感多输出的启动波形，不可能完全像单输出那样“各管各的”。**
