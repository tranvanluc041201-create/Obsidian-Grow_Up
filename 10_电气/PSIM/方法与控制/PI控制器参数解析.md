# PI 控制器参数解析

*面向 PSIM / LTspice 实现的参数理解*

---

## 一、两种常见表达形式

### 1. 并联式

```math
G(s) = K_p + \frac{K_i}{s}
```

这里：

- `Kp`：比例系数
- `Ki`：积分系数

这种写法直观，适合理解“比例项 + 积分项”的叠加。

### 2. 标准式 / 理想式

工业界和很多软件模块更常见的是：

```math
G(s) = K \cdot \left(1 + \frac{1}{T_i s}\right)
```

这里：

- `K`：总增益
- `Ti`：积分时间常数

---

## 二、两种形式之间怎么换算

把标准式展开：

```math
G(s)=K+\frac{K}{T_i s}
```

于是可以得到：

| 对应项 | 关系 |
|--------|------|
| 比例项 | `K = Kp` |
| 积分项 | `Ki = K / Ti` |
| 时间常数 | `Ti = K / Ki = Kp / Ki` |

---

## 三、在 PSIM 中怎么设置

PSIM 的 `PI` 模块更接近标准式，所以你在参数里通常会看到：

- `Gain`
- `Time Constant`

对应关系可以理解成：

| PSIM 参数 | 对应含义 |
|-----------|----------|
| `Gain` | `Kp` |
| `Time Constant` | `Ti = Kp / Ki` |

所以如果你手里的是并联式参数 `Kp`、`Ki`，要先换算成 `Ti`。

---

## 四、在 LTspice 中怎么实现

### 方案 1：直接用一个行为源

```spice
B1 out 0 V=Kp*V(error)+Ki*idt(V(error))
```

这种写法紧凑，适合快速搭建。

### 方案 2：把比例项和积分项拆开

```spice
B2 p_out 0 V=Kp*V(error)
B3 i_out 0 V=Ki*idt(V(error))
```

这样更容易单独观察比例项和积分项的作用。

---

## 五、两个最重要的注意点

### 1. 抗积分饱和

如果误差长期很大，积分项会一直累积，造成：

- 控制量长期贴边
- 恢复很慢
- 启动超调更明显

所以实际系统中要考虑：

- 积分限幅
- 输出限幅
- 必要时做抗饱和设计

### 2. 量纲一致

这在你当前项目里特别关键。

如果：

- `PI` 输出范围是 `0..5`
- 但 `ramp` 是 `0..1`

比较器就会失真。

所以必须保证：

- `Limiter` 输出
- `ramp`
- 最终占空比指令

这些量处在同一个比较范围里。

---

## 六、这篇在当前项目里的作用

对你现在的 `SITO` 项目来说，PI 参数不是孤立的一组数字，而是这条链的一部分：

`误差 → PI → 限幅 → ramp 比较 → PWM`

所以看 `PI` 时，不要只看参数本身，还要同时看：

1. 误差极性对不对
2. 限幅范围对不对
3. `ramp` 量纲对不对
