# 电气学习系统使用说明

*系统课补骨架 + 项目经验沉淀*

---

## 日常流程

### 早上

1. 打开 `[[10_电气/元信息/今日入口]]`
2. 确定今天主任务：
   - 课程复盘
   - 项目推进
   - 基础补充
   - 仿真工具练习

### 进行中

**如果今天做的是课程复盘：**

1. 先做课后回忆
2. 再做主线重构
3. 再选一个单点验证
4. 最后只沉淀一张卡

**如果今天做的是 PSIM 项目：**

1. 先看项目目标
2. 再看关键波形和参考点
3. 再做估算与仿真验证
4. 最后记录问题、现象和结论

**如果今天做的是基础补充：**

1. 先补理论
2. 再映射到 `SITO` 项目经验
3. 再决定是沉到基础区还是方法区

### 晚上（15 分钟）

1. 把随手记整理进正确目录
2. 如果今天踩了新坑，更新 `PSIM/错误类型`
3. 如果今天抽出了新方法，更新 `PSIM/方法与控制`
4. 如果今天推进了项目，补到 `SITO-单电感三输出Buck`

## PSIM → LTspice 快速对照

| PSIM 元件/模块 | LTspice 对应方式 | 说明 |
|----------------|------------------|------|
| Voltage Sensor | 电压探针 | 直接看节点电压 |
| Current Sensor | 电流探针 | 直接看器件支路电流 |
| PI Controller | `B-source` | 用行为源实现比例 + 积分 |
| Limiter | `limit()` 函数 | 把输出压到合理范围 |
| Comparator | `B-source` 比较 | 例如 `V=(V(a)>V(b))?1:0` |
| Ramp | `PULSE` 或自建波形 | 频率和幅值要匹配 |
| Controlled Switch | `SW` | 用 `.model VSwitch` |
| MOSFET | `nmos` / `pmos` | 注意阈值和模型参数 |

### 三个核心差异

1. **接地**
   - PSIM：黑点 `GND`
   - LTspice：必须显式放 `GND`

2. **控制块表达方式**
   - PSIM：偏模块化、拖拽式
   - LTspice：偏方程化、更手工

3. **仿真习惯**
   - PSIM：时域开关仿真更直接
   - LTspice：`.tran / .op / .ac` 更灵活

## 两条学习主线

### 系统课主线

适用：

- 电路分析基础课
- 控制 / 器件 / 拓扑课程视频

默认流程：

1. 课后回忆
2. 主线重构
3. 单点验证
4. 单卡沉淀
5. 次日回看

补充：

- 基础电路课优先手工估算和最小例题
- 不要默认先上仿真

### 项目主线

适用：

- `PSIM` / `LTspice`
- `SITO` 项目推进
- 排错和方法沉淀

## 从 SITO 项目反推理论

这是你当前项目线最有效的学习路径：

1. 先用 `SITO` 项目拿到真实经验
2. 再把经验反推到控制、电感、电容、PWM、参考点、驱动
3. 再把可复用内容抽成方法和错误类型
