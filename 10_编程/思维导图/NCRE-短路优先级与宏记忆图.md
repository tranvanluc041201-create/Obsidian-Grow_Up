# 短路 / 优先级 / 宏

## 两组核心对比
- 看见 =
    - 先想“这是赋值”
    - 左边的变量会被改
- 看见 ==
    - 先想“这是比较”
    - 结果只会是真或假
- 看见 &
    - 先想“按位运算”
    - 左右两边都会算
- 看见 |
    - 先想“按位运算”
    - 左右两边都会算
- 宏 vs 函数
    - 宏：直接替换进去再算
    - 函数：先算实参，再调用函数

## 优先级抓手
- a = b == c
    - 先算 b == c
    - 再把结果赋给 a
- a && b || c
    - 先看左半 a && b
    - 再决定右半要不要算
- !a && b
    - 先把 a 取反
    - 再看 b 会不会执行
- a + b > c
    - 先算加法
    - 再做比较

## 候选易混点
- p&&*p
    - 先看 p 是不是空指针(NULL被看为0)
    - p 非空，才会继续算 *p
    - 这就是“先判地址，再取值”
- 宏参数
    - 参数要加括号
    - 整体也要加括号
- 宏实参带 `++`
    - 可能被展开多次
- 宏末尾
    - 一般不加分号

## 做题抓手
- 先看括号
    - 有没有人为改变顺序
- 再看优先级
    - 到底谁先算
- 再看短路
    - 右边会不会执行
- 最后看宏
    - 是真算函数
    - 还是只做文本替换
