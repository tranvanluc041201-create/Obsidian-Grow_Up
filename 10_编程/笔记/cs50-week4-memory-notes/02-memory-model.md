# 2）内存模型：RAM / 栈 / 堆 / 文件

## 🎯 一句话结论

> 学内存模型，核心只看两件事：数据现在放哪，什么时候失效。

## 📌 这节只解决什么

- 分清 RAM 和文件不是一回事
- 分清栈变量和堆内存不是一回事
- 看到一段代码时，先判断数据的存放位置和生命周期

## ✅ 最小正确写法

```c
int x = 10;                  // 栈
int *p = malloc(sizeof(int)); // 堆
*p = 20;
free(p);
```

## 🔁 代码桥

- 局部变量
  - 一般在栈上
  - 函数结束后通常失效
- `malloc`
  - 向堆申请空间
  - 不会自动释放
- 文件
  - 存在磁盘上
  - 程序结束后还在

## ⚠️ 易混点

| A | B | 怎么区分 |
|---|---|---|
| RAM | 文件 | RAM 是运行时内存；文件在磁盘上 |
| 栈 | 堆 | 栈多半跟作用域走；堆要手动 `free` |
| 变量值 | 地址 | 值是内容；地址是位置 |

## 💥 高频坑

- 以为程序结束后局部变量还能保留
- 以为 `malloc` 的内存会自动释放
- 把“文件里有数据”和“内存里有数据”混成一回事
