# 3）指针 & NULL（重点：你薄弱点）
### 3.2 指针在内存里长什么样（ASCII）

```text
[Stack Snapshot - 示意]
Address      Value            Variable
0x1000       (n 的字节...)    n = 50
0x2000       0x1000           p = &n   （p 里存的是地址，不是 50）--**地址本身也有地址,其值对应所取变量的地址**
```

### 3.3 NULL 是什么（必须讲透）

* **NULL = 空指针**：*表示“这个指针不指向任何有效对象”*
* 任何对 NULL 的解引用（`*p` / `p[i]` / `p->field`）都是高概率崩溃/未定义行为

> 规则：**解引用之前，必须确保指针指向合法对象**（新手阶段：先会判 NULL）
> 原因:NULL既不可读也不可写,任何关于NULL的读写行为都是未定义行为(UB)

### 3.4 NULL vs '\0'（最常混）

* `NULL`：空指针（给指针用）
* `'\0'`：空字符（字符串结束符，存在字符数组里）

```text
(1) NULL 指针：指针变量里放“无效地址”
Address      Value         Variable
0x2000       *0x00000000*    p = NULL

(2) '\0'：字符串内容的一部分
Address      Value   Char
0x3000       0x48    'H'
0x3001       0x49    'I'
0x3002       0x00    '\0'
```

### 3.5 两种“炸点”对比：NULL vs 未初始化指针

#### A）malloc 失败返回 NULL（必须判）

```c
int *a = malloc(3 * sizeof(int));
if (a == NULL) {
    return 1;
}
/将a指向申请内存的空间,如果内存不够会返回NULL表示申请失败
```

#### B）未初始化指针（更阴险：不是 NULL，是垃圾地址）

```c
char *s;            // 🚨 未初始化：垃圾地址
scanf("%s", s);     // 往垃圾地址写：高概率炸
/s 可能指向一些电脑自身关键数据,一旦修改电脑容易崩溃
```

安全做法：

* 用数组缓冲区：`char s[100]; scanf("%99s", s);`
* 或 malloc：`char *s = malloc(100); ... free(s);`

**我的理解**：

* 我理解的 `NULL`：表示“当前没有指向任何合法对象”，可以安全地表示空，但不能解引用
* 我理解的 `'\0'`：字符串结束符，存在字符数组里，值是 0，但它不是空指针
* 我最怕的场景：定义了指针却没初始化，就直接读写 `*p`
