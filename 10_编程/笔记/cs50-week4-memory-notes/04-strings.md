# 4）字符串（C String）

*C 字符串的本质：字符数组 + '\0'*

---

# 🎯 一句话结论

**C 字符串是 `char` 数组，以 `'\0'` 结尾**。比较内容用 `strcmp()`，不能用 `==`（`==` 比较的是地址）。

---

## 💥 问题场景

### 场景 1：用 == 比较字符串

**错误的代码**：
```c
char *s = "hello";
char *t = "hello";
if (s == t) {  // 🚨 比较的是地址，不是内容！
    printf("相同\n");
}
```

**会发生什么**：
可能输出"相同"（如果编译器优化让两个字符串共用内存），也可能不输出。结果是**未定义行为**。

---

### 场景 2：忘记 '\0' 导致越界

**错误的代码**：
```c
char s[3];
s[0] = 'h';
s[1] = 'i';
// 忘记 s[2] = '\0'
printf("%s", s);  // 🚨 越界读，直到遇到随机 '\0'
```

**会发生什么**：
输出 "hi" + 一堆乱码，因为 `printf` 会一直读到内存中遇到的第一个 `'\0'`。

---

## 🤔 为什么（通俗理解）

### C 字符串的本质

```
字符串 "HI" 在内存中：
┌─────┬─────┬─────┐
│ 'H' │ 'I' │ '\0' │
│ 72  │ 73  │  0   │  <- ASCII 码
└─────┴─────┴─────┘
  ↑                ↑
起始地址          结束标志
```

**类比**：
- 字符串 = 火车（字符是车厢，`'\0'` 是终点站牌）
- `printf` 从车头开始读，看到终点站牌才停

### 为什么 == 不行？

```c
char *s = "hello";  // s 存的是字符串的地址，比如 0x1000
char *t = "hello";  // t 存的可能也是 0x1000（编译器优化）
                    // 也可能是 0x2000（不同地址）

s == t  // 比较的是 0x1000 == 0x2000？不一定！
```

**核心问题**：`==` 比较的是指针变量的值（地址），不是字符串的内容。

---

## ✅ 正确写法

- [ ] 复习：字符串比较 #知识点复盘 ⏫ 📅 2026-02-19 ⏳ 2026-02-19 

```c
char *s = "apple";
char *t = "banana";

// ❌ 错误
if (s == t) { ... }

// ✅ 正确
int result = strcmp(s, t);//本质是字符串里的字符ASCII码的大小比较
if (result == 0) {
    printf("相同\n");
} else if (result < 0) {
    printf("s 在 t 前面（字典序）\n");
} else {
    printf("s 在 t 后面\n");
}
```

- [ ] 复习：字符串初始化 #知识点复盘 ⏫ 📅 2026-02-19 ⏳ 2026-02-19 

```c
// ✅ 方法 1：字符串字面量（自动加 '\0'）
char s[] = "hi";  // s = {'h', 'i', '\0'}

// ✅ 方法 2：显式指定大小
char s[10] = "hi";  // 前3字节是 "hi\0"，后面是 0

// ✅ 方法 3：手动设置（容易忘 '\0'）
char s[3];
s[0] = 'h';
s[1] = 'i';
s[2] = '\0';  // 别忘了！
```

### 安全复制字符串

```c
char dest[10];
char *src = "hello";

// ❌ 危险：可能溢出
strcpy(dest, src);

// ✅ 安全：指定最大复制长度
strncpy(dest, src, sizeof(dest) - 1);
dest[sizeof(dest) - 1] = '\0';  // 确保结尾
```

**关键检查点**：
- [x] 比较字符串用 `strcmp`，不是 `==` ✅ 2026-02-18
- [x] 字符数组确保有足够空间放 `'\0'` ✅ 2026-02-18
- [ ] 字符串复制用 `strncpy`，防止溢出
- [x] 手动设置字符后，记得加 `'\0'` ✅ 2026-02-18

## 🔍 高阶视角

### 字符串常量 vs 字符数组

```c
char *s = "hello";     // 字符串常量，只读！
char t[] = "hello";    // 字符数组，可修改

s[0] = 'H';  // 🚨 未定义行为（可能崩溃）
t[0] = 'H';  // ✅ 正常，t 变成 "Hello"
```

```c
// 字符串常量 - 只读！
char *s = "hello";   // "hello"在只读数据段
s[0] = 'H';          // ❌ 未定义行为（可能崩溃）

// 字符数组 - 可修改！
char t[] = "hello";  // 数组在栈上，内容是拷贝过来的
t[0] = 'H';          // ✅ 正常，t变成"Hello"

理解:字符串本身是不可以被修改的(通过字符串的地址),但是字符串存入字符数组是可以被修改的
```
**工程启示**：
- 字符串常量放在只读内存区
- 需要修改时用数组 `char s[] = "..."`
- 只读指针用 `const char *`
