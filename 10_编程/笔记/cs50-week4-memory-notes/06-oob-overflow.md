# 6）越界与溢出：先看对象边界

## 🎯 一句话结论

> 越界的本质是：你碰到了不属于这个对象的内存。

## 📌 这节只解决什么

- 什么叫越界读、越界写
- 为什么字符串最容易越界
- 做题时先从哪里检查边界

## ✅ 最小正确写法

```c
char buf[5];
strcpy(buf, "ABCD");
```

这里能放下，是因为：

- `A B C D` 四个可见字符
- 再加一个结尾 `'\0'`

## 🔁 代码桥

### 先看对象有多大

- 数组有几个元素
- 每个元素多大

### 再看你要写多少

- 可见内容有多少
- 有没有把 `'\0'` 算进去

### 最后看循环边界

- 新手阶段优先警惕 `<=`
- 看到长度变量，就先代值检查一遍

## ⚠️ 易混点

### “差一个字节” vs “问题不大”

- 前者：很多人主观上觉得无所谓
- 后者：在 C 里已经是未定义行为
- 一句话区分：边界一旦越过，哪怕只 1 个字节，也已经错了

## 💥 高频坑

- `char buf[4]` 却想放 `"ABCD"`
- 循环写成 `i <= n`
- `malloc(n)` 后却按 `n` 个 `int` 来用
