# 6）越界 / 溢出（按“对象边界”理解，最清晰）

## 🎯 一句话结论

> 所谓越界，本质上就是：**你访问了不属于这个对象的那一段内存**。

---

## 📌 这节课到底在解决什么

- 主题：越界读、越界写、缓冲区溢出、整数溢出
- 典型场景：数组长度不够、字符串没留 `'\0'`、循环写多一位
- 如果不会，会错在哪里：程序表面能跑，但数据已经悄悄被写坏

---

## 🤔 核心概念

- **越界读**：读了对象边界外的数据
- **越界写**：写到了对象边界外的数据
- **缓冲区溢出**：最常见的越界写
- **整数溢出**：值先算错，再导致长度或索引算错

### 最小例子
```c
char buf[4];   /* 最多放 3 个字符 + '\0' */
strcpy(buf, "ABCD");  /* ❌ 实际要写 5 个字节 */
```

---

## ✅ 正确做法

- 写字符串时先数清：内容长度 + `'\0'`
- 循环边界优先写 `<`，少写 `<=`
- 看到 `malloc(n * sizeof(type))` 时先检查 `n` 对不对

### 改哪里
- 原来错在：只数了可见字符，没数结束符
- 正确应该改成：先算对象真实边界，再决定数组大小
- 为什么这样改：C 不会替你拦住边界外访问

---

## ⚠️ 常见坑

- `char buf[4]` 却想放 `"ABCD"`
- `for (i = 0; i <= n; i++)`
- `malloc(n)` 却按 `n` 个 `int` 使用

---
