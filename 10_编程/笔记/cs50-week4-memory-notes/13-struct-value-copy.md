# 13）结构体赋值与传参：成员复制、指针字段与值传递

## 🎯 一句话结论

> 结构体赋值、传参、返回值，默认先按“成员复制 / 值传递”理解；想改原对象，再考虑传指针。

## 📌 这节只解决什么

- 看见 `b = a`，知道这通常是复制，不是共享
- 看见 `func(p)`，先判断传的是结构体本体还是结构体指针
- 分清 `.` 和 `->` 的使用场景

## ✅ 最小正确写法

```c
typedef struct {
    int x;
    int y;
} Point;

Point a = {10, 20};
Point b = a;
b.x = 99;
```

此时：

- `a.x` 还是 `10`
- `b.x` 变成 `99`

## 🔁 代码桥

- 想“复制一个结构体”
  - 直接写：`b = a;`
- 想“在函数里改原结构体”
  - 传指针：`func(&p);`
- 看到 `p->x`
  - 先判断：`p` 是不是结构体指针

## ⚠️ 易混点

| A | B | 怎么区分 |
|---|---|---|
| `Point p` | `Point *p` | 前者是结构体变量；后者是指向结构体的指针 |
| `p.x` | `p->x` | 变量本体用 `.`；结构体指针用 `->` |
| `move(p)` | `move(&p)` | 前者通常改副本；后者才有机会改原对象 |

## 一个最常见的坑

```c
void move(Point p) {
    p.x = 100;
}
```

这段代码改不到外面的原变量，因为：

- 函数参数 `p` 是一份副本
- 你改的是副本，不是外面的原对象

如果要改原变量：

```c
void move(Point *p) {
    p->x = 100;
}
```

## 💥 高频坑

- 以为 `b = a` 后，两者会共用一份结构体数据
- 以为函数参数里的结构体字段改了，外面一定会跟着变
  
## 特别补充：有指针字段时要更小心

```c
typedef struct {
    char *name;
} Person;
```

如果 `Person a; Person b = a;`，那么：

- 结构体本身仍然是成员复制
- 但 `name` 这个字段里放的是地址
- 所以复制后，两个结构体里的 `name` 可能指向同一块数据(值(地址)传递)

这时危险点不在“结构体不会复制”，而在“指针字段复制的是地址”。
