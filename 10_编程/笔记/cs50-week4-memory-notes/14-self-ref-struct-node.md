# 14）自引用结构体与 Node 结点

## 🎯 一句话结论

> `next` 里放的是“下一个结点的地址”，不是“下一个结点本体”。

## 📌 这节只解决什么

- 什么是“链表（linked list）”
- 什么是“结点（Node）”
- 为什么能写 `struct Node *next`
- 为什么不能写 `struct Node next`

## 链表是什么

先把链表看成一串“小格子”。

一个“小格子”，就是一个结点（Node）。

每个结点只回答两件事：

1. 我自己存什么数据  
2. 下一个结点在哪里  

### 和数组的关键区别

数组靠“连续位置”组织。  
链表靠“地址连接”组织。

### 先记两个结果

- 插入、删除更灵活
- 不能像数组那样按下标直接跳过去

## ✅ 最小正确写法

```c
struct Node {
    int data;
    struct Node *next;
};
```

## 🔁 代码桥

### 先看 `Node`

- `Node` / 结点 = 链表里的一个小格子
- 这个格子的作用有两个：
  - 放自己的数据
  - 连到下一个结点

### 再看 `int data`

- `data` 表示“这个结点自己的数据”
- 这里写 `int`，只是举最常见的例子
- 以后也可以换成别的类型

### 最后看 `struct Node *next`

- `next` 里放的是“下一个结点的地址”
- 不是下一个结点本体
- 之所以能这样写，是因为这里存的是指针
- 指针大小固定，所以编译器能先处理这条成员声明
- 最后一个结点后面没有了，通常写 `NULL`

## ⚠️ 易混点

### `struct Node *next` vs `struct Node next`

- 前者：成员里只放一个地址，所以可以写
- 后者：成员里要再放一个完整 `Node`
- 这样会一层套一层，结构体大小就算不出来

## 💥 高频坑

- 把 `next` 当成“下一个结点本体”
- 结构体定义末尾漏分号：`};`

## 一个最小串起来的例子

```c
struct Node {
    int data;
    struct Node *next;
};

struct Node a = {10, NULL};
struct Node b = {20, NULL};

a.next = &b;
```

现在的意思是：

- `a` 这个结点里存着 `10`
- `a.next` 指向 `b`
- `b` 是后一个结点
- `b.next = NULL` 表示到头了

## 为什么最后一个结点写 `NULL`

- `NULL` 的意思是：这里没有再指向任何有效结点
- 所以遍历链表时，常用它当“到头了”的标记
- 看到 `while (p != NULL)`，通常可以直接理解成：
  - 只要还没走到链表末尾
  - 就继续往后走
