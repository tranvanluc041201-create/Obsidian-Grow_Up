# 16）内存对齐与结构体大小

## 🎯 一句话结论

> 结构体大小不一定等于“成员大小直接相加”；编译器常会插入填充字节，让成员按对齐要求摆放。

## 📌 这节只解决什么

- 为什么结构体大小会比想象中大
- 怎么快速估一个结构体大小
- 什么叫“尾部填充”

## ✅ 最小正确写法

```c
struct A {
    char c;
    int x;
};
```

常见 64 位环境下：

- `char` 占 1
- `int` 常占 4
- `sizeof(struct A)` 常是 `8`，不是 `5`

## 🔁 代码桥

- 对齐
  - 编译器让成员放在更适合 CPU 读取的位置
- 填充字节
  - 成员之间可能会塞空位
- 尾部填充
  - 最后还可能补齐到“最大对齐要求”的倍数

## ⚠️ 易混点

| A | B | 怎么区分 |
|---|---|---|
| 成员大小相加 | `sizeof(struct)` | 两者不一定相等 |
| 成员间填充 | 尾部填充 | 前者在成员之间；后者在结构体末尾 |
| `char` 小 | 结构体就一定小 | 不一定，后面成员可能触发填充 |

## 💥 高频坑

- 直接把结构体大小当成成员大小求和
- 忽略最大对齐成员对总大小的影响
- 把不同平台的结果当成绝对固定
