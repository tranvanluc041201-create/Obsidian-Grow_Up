# 17）二级指针：指针的指针

## 🎯 一句话结论

> `int **pp` 不是更神秘的变量，它只是“存放 `int *` 地址的指针”。

## 📌 这节只解决什么

- `p` 和 `pp` 各自存什么
- `*p`、`**pp` 各取到什么
- 函数为什么有时要传二级指针

## ✅ 最小正确写法

```c
int x = 10;
int *p = &x;
int **pp = &p;
```

## 🔁 代码桥

- `p`
  - 存 `x` 的地址
- `pp`
  - 存 `p` 的地址
- `*p`
  - 取到 `x`
- `*pp`
  - 取到 `p`
- `**pp`
  - 最终取到 `x`

## ⚠️ 易混点

| A | B | 怎么区分 |
|---|---|---|
| `p` | `*p` | 前者是地址；后者是地址里的值 |
| `pp` | `*pp` | 前者是“指针的地址”；后者是一级指针 |
| 改 `p` 指向 | 改 `*p` 的值 | 前者常需要二级指针；后者一级指针就够 |

## 什么时候会用到二级指针

- 函数里想改“外面那个指针变量本身”
- 例如：让函数帮你 `malloc` 并把新地址写回去

```c
void alloc_int(int **pp) {
    *pp = malloc(sizeof(int));
}
```

## 💥 高频坑

- 把 `**pp` 当成一种新类型去死记
- 只会背星号个数，不会顺着“地址链”往回看
- 函数想改外面指针，却只传了一级指针
