# 最小公倍数与最大公约数的判断

## 🎯 一句话结论

> 最大公约数先用辗转相除法，最小公倍数再用 `a * b / gcd(a, b)`，不要反过来硬猜。

---

## 📌 这节课到底在解决什么

- 主题：整数的最大公约数与最小公倍数
- 典型场景：循环、取模、数值题、程序设计题
- 如果不会，会错在哪里：把“第一个除尽的数”看错，或者把最小公倍数公式写反

---

## 🤔 核心概念

### 1. 为什么这么做
- 最大公约数本质上是在不断缩小问题规模
- 最小公倍数本质上是在“两个数共有部分只保留一份”

### 2. 代码里它长什么样
```c
int gcd(int a, int b)
{
    while (b != 0) {
        int temp = a % b;
        a = b;
        b = temp;
    }
    return a;
}

int lcm(int a, int b)
{
    return a / gcd(a, b) * b;
}
```

### 3. 最容易混淆的点
| 概念A | 概念B | 区别 |
|------|------|------|
| 最大公约数 | 最小公倍数 | 一个找“共有最大因子”，一个找“公共倍数里最小的” |
| `%` 取余 | `/` 除法 | 辗转相除法核心用 `%`，不是只做除法 |

---

## ✅ 正确做法

### 推荐写法
```c
/* 例：求 78 和 91 的 gcd / lcm */
int g = gcd(78, 91);     /* 13 */
int l = 78 / g * 91;     /* 546 */
```

### 改哪里
- 原来错在：把“第一个能整除的数”记成了最后一次余数
- 正确应该改成：辗转相除直到余数为 0，此时除数才是 gcd
- 为什么这样改：算法停止时，当前除数就是两个数共同保留下来的最大因子

### 怎么验证
- [ ] 我能手算 `91 % 78 = 13`、`78 % 13 = 0`
- [ ] 我能说出为什么 `lcm = a / gcd * b`
- [ ] 我能写出 `gcd` 的 while 版本

---

## ⚠️ 常见坑

- 把“最后一个非 0 余数”与“最后一个除数”混淆
- 直接写 `a * b / gcd`，大数时可能先溢出
- 不处理负数或 0 的情况
