# 1.1 循环或判断后的分号截断

## 🎯 题目 / 场景

for循环后多余的分号导致逻辑错误：

```c
for(int i=0; i<10; i++); // ❌ 这里的 ; 结束了循环体
{
    printf("Running...\n"); // 结果：只打印一次，而不是10次
}
```

---

## 💥 发生了什么

- 预期：打印10次 "Running..."
- 实际：只打印1次
- 原因：`for` 循环体是**空语句**（只有那个分号），`printf` 在循环外执行

---

## 🤔 我当时怎么想的

> "分号只是结束符，写在哪都一样..."
> 
> **盲区**：没意识到分号本身就是一条语句（空语句），`for` 会把空语句当成循环体。

---

## 🔍 根因分析

### 表面原因
> for循环后的分号让循环体变成空语句

### 深层原理
> **空语句（Empty Statement）**：C语言允许单独的分号作为语句，表示"什么都不做"。
> 
> `for(init; cond; update);` 这个结构里，最后的 `;` 就是循环体。
> 
> 循环执行10次空语句，结束后才执行下面的代码块。

### 识别信号
> 凡是控制结构（`if`/`for`/`while`）后面直接跟分号的，99%是错误。
> 例外：故意用空语句，如 `while(*p++);`（找字符串结尾）。

---

## ✅ 正确规则与修正

```c
// ✅ 正确写法1：去掉分号
for(int i=0; i<10; i++) {
    printf("Running...\n");
}

// ✅ 正确写法2：如果确实想用空语句（如只想要i++的效果）
for(int i=0; i<10; i++) {
    ; // 空语句写在花括号内，意图明确
}
```

---

## 🔁 一题多变 / 横向关联（视情况补充）

### 同类变式
- **if语句后的分号**
  ```c
  if (x > 0);      // ❌ 空语句
      printf("x>0"); // 这行无条件执行
  
  // ✅ 正确
  if (x > 0) {
      printf("x>0");
  }
  ```

- **while后的分号导致死循环**
  ```c
  while (x > 0); {  // ❌ 空语句循环，x不变，死循环
      x--;
  }
  ```
