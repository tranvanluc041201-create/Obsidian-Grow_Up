# 1.2 宏定义运算优先级

## 🎯 题目 / 场景

宏定义表达式缺少括号导致运算优先级错误：

```c
#define A 2
#define B A+3  // ❌ 没有加括号

int result = B * 5;  // 预期 25，实际 17
```

---

## 💥 发生了什么

- **预期**：`(2+3)*5 = 25`
- **实际**：`2+3*5 = 17`
- **原因**：宏是**纯文本替换**，`B` 被替换为 `2+3`，不是 `(2+3)`

**展开过程**：
```c
int result = B * 5;
// 展开：
int result = 2+3 * 5;  // 先算 3*5=15，再 2+15=17
```

---

## 🤔 我当时怎么想的

> "宏定义就是简单的替换，运算时会自动按优先级计算..."
> "A+3 是一个整体，应该优先计算..."
> **盲区**：不知道宏只是**纯文本替换**，不做任何运算或优先级调整。

---

## 🔍 根因分析

### 表面原因
> 宏表达式没有加括号，导致展开后运算优先级与预期不符

### 深层原理

**宏的本质**：
- 预处理阶段**纯文本替换**
- 不进行任何计算
- 不检查语法（除了基本的括号匹配）
- 完全信任程序员写的替换文本

**对比**：
```c
// 宏
#define B A+3
int x = B * 5;  // 展开：A+3 * 5

// 常量（C++ const）
const int B = A+3;  // 编译期计算，B=5
int x = B * 5;      // 25
```

### 识别信号
> **宏表达式必须全程加括号**：参数加括号，整体加括号。

---

## ✅ 正确规则与修正

```c
// ✅ 正确写法：全程加括号
#define A 2
#define B (A+3)  // 整体加括号

int result = B * 5;  // 展开：(2+3) * 5 = 25 ✓

// ✅ 更复杂的例子
#define MAX(a, b) ((a) > (b) ? (a) : (b))  // 参数和整体都加括号
#define SQUARE(x) ((x) * (x))               // 参数加括号
```

## 🔁 一题多变 / 横向关联（视情况补充）

### 同类变式
- **多层宏嵌套**
  ```c
  #define A 2
  #define B A+3
  #define C B*2  // C = 2+3*2 = 8，不是 10！
  
  // ✅ 全部加括号
  #define A 2
  #define B (A+3)
  #define C (B*2)  // C = (2+3)*2 = 10
  ```

### 工程启示
> **宏定义编码规范**：

> 1. 宏表达式**必须**加括号（整体）
> 2. 宏参数**必须**加括号（防止传入表达式）
> 3. 复杂的计算用 `const` 或 `static inline` 代替宏
> **现代C推荐**：
> - 能用 `const int` 就不用 `#define`
> - 能用 `static inline` 函数就不用宏函数
