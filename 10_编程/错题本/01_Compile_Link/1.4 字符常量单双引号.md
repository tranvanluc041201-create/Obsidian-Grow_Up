# 1.4 字符常量单双引号

## 🎯 题目 / 场景

字符比较时误用双引号：

```c
char c = 'A';
if (c == "A") {  // ❌ 错误！
    // ...
}
```

---

## 💥 发生了什么

- **预期**：判断字符 c 是否等于 'A'
- **实际**：比较字符和字符串地址
- **结果**：逻辑错误（可能永远不成立）

**区别**：
```c
'A'   // 字符常量，类型是 int，值为 65（ASCII码）
"A"   // 字符串常量，类型是 char*，指向字符串的地址
```

---

## 🤔 我当时怎么想的

> "双引号也是表示字符..."
> "单双引号应该差不多..."
> **盲区**：不知道单引号是字符，双引号是字符串（地址）。

---

## 🔍 根因分析

### 表面原因
> 误用双引号字符串代替单引号字符

### 深层原理

**C语言中的引号**：
| 写法 | 类型 | 值 | 含义 |
|------|------|-----|------|
| `'A'` | int | 65 | 字符A的ASCII码 |
| `"A"` | char[2] | 地址 | 字符串"A\0" |
| `'AB'` | int | 多字符 | 平台相关，避免使用 |

**内存布局**：
```c
"A" 在内存中：['A']['\0']  // 2字节，有结束符
```

### 识别信号
> **字符比较用单引号 `'c'`，字符串用双引号 `"str"`**。

---

## ✅ 正确规则与修正

```c
// ❌ 错误
if (c == "A")

// ✅ 正确
if (c == 'A')

// ✅ 字符串比较用 strcmp
char str[] = "Hello";
if (strcmp(str, "Hello") == 0)  // 字符串比较
```

---

## 🔁 一题多变 / 横向关联（视情况补充）

### 同类变式
- **多字符常量（危险）**
  ```c
  int x = 'ABCD';  // 平台相关，避免使用
  // 在某些编译器上等价于：('A'<<24) | ('B'<<16) | ('C'<<8) | 'D'
  ```

### 工程启示
> **编码规范**：
> - 字符常量：单引号 `'c'`
> - 字符串常量：双引号 `"str"`
> - 绝不混用
> **常见场景**：
> - `switch case`：`case 'A':` 不是 `case "A":`
> - 字符数组初始化：`char s[] = "hi";` 不是 `char s[] = {'h','i'};`（缺少\0）
