# 函数声明与定义混淆

## 🎯 题目 / 场景

函数声明与定义混淆，导致编译错误：

```c
// ❌ 在.h文件中写实现（带函数体）
// math_utils.h
int multiply(int x, int y) {  // 错误！头文件不应该有函数体
    return x * y;
}

// ❌ 在.c文件中只写声明（带分号没有函数体）
// math_utils.c
int multiply(int x, int y);  // 错误！源文件应该有实现
```

**编译错误**：
- `undefined reference to 'multiply'`
- `multiple definition of 'multiply'`

---

## 💥 发生了什么

- **预期**：多文件程序正确编译链接
- **实际**：编译或链接失败
- **原因**：不理解**声明**和**定义**的区别

## 🔍 根因分析

### 表面原因
> 混淆了声明和定义的位置

### 深层原理

| 类型 | 特征 | 位置 | 作用 |
|------|------|------|------|
| **声明** | 带分号`;`，无函数体`{}` | `.h` 文件 | 告诉编译器"有这个函数" |
| **定义** | 无分号，有函数体`{}` | `.c` 文件 | 真正的实现代码 |

**编译链接过程**：
1. **编译阶段**：每个`.c`文件独立编译，需要知道函数签名（声明）
2. **链接阶段**：链接器找函数的实现（定义），如果没有或有多份就报错

### 识别信号
> **头文件放声明，源文件放定义**。

---

## ✅ 正确规则与修正

```c
// ============ math_utils.h ============
#ifndef MATH_UTILS_H
#define MATH_UTILS_H

// 声明：只告诉编译器函数签名
int multiply(int x, int y);

#endif

// ============ math_utils.c ============
#include "math_utils.h"

// 定义：真正的实现
int multiply(int x, int y) {
    return x * y;
}

// ============ main.c ============
#include <stdio.h>
#include "math_utils.h"

int main(void) {
    int result = multiply(5, 3);  // 编译器通过.h知道签名
    printf("结果: %d\n", result);
    return 0;
}
```

## 🔁 一题多变 / 横向关联（视情况补充）

### 同类变式
- **变量声明vs定义**
  ```c
  extern int x;  // 声明（告诉编译器x在其他地方定义）
  int x = 10;    // 定义（分配存储空间）
  ```

### 工程启示
> **头文件保护**：
> - 用`#ifndef/#define/#endif`防止重复包含
> - 或者用`#pragma once`（非标准但广泛支持）
> **链接错误排查**：
> - `undefined reference` → 缺少定义（没链接对应的.o文件）
> - `multiple definition` → 重复定义（头文件里写了函数体）

|------|------|------|
| **.h文件** | 声明 | 被多个.c包含，不能重复定义 |
| **.c文件** | 定义 | 只编译一次，链接时使用 |
