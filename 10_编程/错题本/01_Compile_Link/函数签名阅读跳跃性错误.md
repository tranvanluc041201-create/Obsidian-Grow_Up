# 函数签名阅读跳跃性错误

## 🎯 题目 / 场景

阅读函数签名时思维跳跃，忽略返回类型：

```c
// 大脑自动过滤了 void，看成 int swap(...)
void swap(int *x, int *y) {
    // 实现...
}
```

**后果**：
- 在void函数里找return语句
- 调用时误以为是int类型

---

## 💥 发生了什么

- **现象**：看完代码后理解错误
- **原因**：跳读习惯，思维定式（默认函数返回int）

> "函数一般都是int返回类型..."
> "扫一眼就知道什么意思..."
> **盲区**：没有逐字阅读，被思维定式误导。

---

## 🔍 根因分析

### 表面原因
> 跳读代码，没有仔细看清返回类型

### 深层原理

**思维定式（Mental Model）**：
```c
// 因为经常写：
int main(void) { ... }

// 所以看到函数就默认是 int 返回类型
void swap(...)  // 大脑自动过滤了 void
```

**C语言历史**：
- C89标准：函数不写返回类型默认是`int`（现在已废弃）
- 现代C：必须显式写返回类型

### 识别信号
> **看见函数三问**：

> 1. 返回什么类型？（void/int/指针？）
> 2. 参数什么类型？（值传递/指针？）
> 3. 要不要 return？（void 不要，其他要）

---

## ✅ 正确规则与修正

```c
// ✅ 慢读法：逐字默念
// "void" - 哦，返回空，不需要 return
// "swap" - 函数名叫 swap
// "(int *x, int *y)" - 两个 int 指针参数

// ✅ TDD式填空：先写签名，再填空
void swap(int *x, int *y) {  // 第1步：抄写签名
    // 第2步：填空实现
    int t = *x;
    *x = *y;
    *y = t;
}  // void函数，没有return
```
