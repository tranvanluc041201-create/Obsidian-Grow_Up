# 2.1 二维数组传参

## 🎯 题目 / 场景

二维数组传参时维度不匹配导致数据错乱：

```c
// 主函数
int a[20][20];
f(a);  // 传递二维数组

// 子函数 - 错误的定义
void f(int a[][3]) {  // ❌ 第二维是3，不是20！
    // 访问 a[i][j] 时位置错乱
}
```

---

## 💥 发生了什么

- **预期**：`a[1][0]` 是第2行第1列
- **实际**：`a[1][0]` 访问到了错误的位置
- **原因**：内存步长不匹配

---

## 🤔 我当时怎么想的

> "只要是个二维数组，子函数里怎么定义都行..."
> "C语言不是不检查数组越界吗，应该没问题..."
> **盲区**：不知道二维数组在内存中是**按行连续存储**的，编译器需要根据第二维计算步长。

---

## 🔍 根因分析

### 表面原因
> 子函数中二维数组的第二维与主函数不一致

### 深层原理

**二维数组的内存布局**：
```
a[0][0] a[0][1] ... a[0][19] a[1][0] a[1][1] ...
└─────────── 第0行 ───────────┘└─────────── 第1行 ───────────┘
```

**步长计算**：
- 主函数 `a[20][20]`：跳一行 = 跳过 20 个 int = 80 字节
- 子函数 `a[][3]`：跳一行 = 跳过 3 个 int = 12 字节

**访问 `a[1][0]` 时**：
- 主函数视角：基地址 + 20×sizeof(int)
- 子函数视角：基地址 + 3×sizeof(int)
- **结果**：访问到了错误的位置！

### 识别信号
> **数组传参退化为指针**，但二维数组需要知道**列数**才能计算行偏移。
> 第一维可以省略（编译器不关心），第二维**必须**正确。

---

## ✅ 正确规则与修正

```c
// ✅ 方法：维度严格一致
void f(int a[][20]) {  // 第二维必须是20！
    // 正确访问
}

```

## 🔁 一题多变 / 横向关联（视情况补充）

### 同类变式
- **三维数组传参**
  ```c
  int a[10][20][30];
  void f(int a[][20][30]);  // 只有第一维可省略
  ```

- **动态二维数组**
  ```c
  // 指针数组方式（每行独立分配）
  int **a = malloc(rows * sizeof(int*));
  for (int i = 0; i < rows; i++) {
      a[i] = malloc(cols * sizeof(int));
  }
  理解:
```
1. a 是 int** 类型
      ↓
2. a 指向一块内存，这块内存里存的是 int*（指针）--数组指针(数组元素为指针)
      ↓
3. 每个 int* 又指向另一块内存（存的是 int 数据）--常熟数组(数组元素为常量)
```
  // 此时传参是 int **，不是 int[][]
  ```

### 工程启示
> **C语言数组传参的陷阱**：

> 1. 一维数组传参退化为指针，丢失长度信息
> 2. 二维数组传参必须保留列数（第二维）
> 3. 建议同时传递维度信息，或使用结构体封装

> **最佳实践**：

> ```c
> typedef struct {
>     int data[20][20];
>     int rows, cols;
> } Matrix;
> ```
