# 2.11 三维数组传参类型匹配

## 🎯 题目 / 场景

三维数组传参时的类型匹配问题：

> "我觉得 a[5][6] 不可以看成 a[0][5][6] 吗"

**背景代码**：
```c
void f(int (*a)[5][6]);  // 期望：指向二维数组的指针

int main() {
    int a[5][6];         // 二维数组
    f(a);                // 能传进去吗？
}
```

## 💥 发生了什么

**直觉**：二维数组多加一个维度就变成三维数组了？

**实际情况**：`a[5][6]` 和 `a[1][5][6]` 是**完全不同的类型**，不能传参

---

> **盲区**：没有理解数组类型的本质，以及函数参数退化的规则。

## 🔍 根因分析

### "盒子套盒子"模型

#### 三维数组 `int b[2][5][6]` 的结构
```
b 是"大盒子"（三维数组）
├── b[0] 是"中盒子"（二维数组 int[5][6]）
│   ├── b[0][0] 是"小盒子"（一维数组 int[6]）
│   ├── b[0][1] 是"小盒子"
│   └── ...
└── b[1] 是另一个"中盒子"
    ├── b[1][0] 是"小盒子"
    └── ...

函数期望：int (*)[5][6] → 指向"中盒子"的指针
传入 b 时：b 退化为 &b[0] → 指向第一个"中盒子" ✓ 匹配！
```

#### 二维数组 `int a[5][6]` 的结构
```
a 是"中盒子"（二维数组）
├── a[0] 是"小盒子"（一维数组 int[6]）
├── a[1] 是"小盒子"
└── ...

函数期望：int (*)[5][6] → 指向"中盒子"的指针
传入 a 时：a 退化为 &a[0] → 指向第一个"小盒子"
           "小盒子"的类型是 int[6]
           但函数期望的是 int[5][6]（中盒子）
           ✗ 不匹配！类型错误！
```

### 类型对比

| 声明 | 实际类型 | 传参时退化为 |
|------|----------|--------------|
| `int b[2][5][6]` | 三维数组 | `int (*)[5][6]` ✓ 匹配 |
| `int a[5][6]` | 二维数组 | `int (*)[6]` ✗ 不匹配！ |

### 为什么不能"看成 a[0][5][6]"？

```c
// 错误理解：
int a[5][6];  // 想看成 a[0][5][6]？

// 问题1：a[0][5][6] 是什么？
// - a[0] 是 int[6] 类型
// - a[0][5][6] 是语法错误！a[0] 不是数组名，不能再用[]

// 问题2：即使强行理解，类型也不匹配
// a 的类型：int (*)[6]（指向含6个int的数组的指针）
// 期望类型：int (*)[5][6]（指向含5×6个int的二维数组的指针）
```

---

## ✅ 正确规则与修正

```c
// ✅ 三维数组传参（正确）
void f(int (*a)[5][6]);   // 期望指向"5行6列二维数组"的指针

int b[2][5][6];           // b退化为 &b[0]，类型是 int (*)[5][6]
f(b);                     // ✓ 匹配！

int c[10][5][6];          // 同样匹配
f(c);                     // ✓ 可以！

// ❌ 二维数组传参（错误）
int a[5][6];              // a退化为 &a[0]，类型是 int (*)[6]
f(a);                     // ✗ 不匹配！期望 int (*)[5][6]，实际 int (*)[6]

// 正确的二维数组传参应该是：
void g(int (*a)[6]);      // 期望指向"6个int数组"的指针
g(a);                     // ✓ 匹配！
```

## 🔁 一题多变 / 横向关联（视情况补充）

### 类型匹配公式

```
函数参数类型：T (*p)[N][M]...
可以传入：T [K][N][M]...（任意K，任意维度N,M...）

函数参数类型：T (*p)[N]
可以传入：T [K][N]（二维数组）
不可以传入：T [N][M]（三维数组）
```

### 记忆口诀

```
多维数组传参看本质
退化成指针少一维
期望指针指向什么
就要传对应维度的数组

int (*p)[5][6] 期望"指向二维数组"
必须传三维数组，退化成"指向二维数组"
传二维数组退化后是"指向一维数组"，不匹配！
```

## 🎯 关键记忆点

| 要点 | 口诀 |
|------|------|
| **三维数组退化** | `T [K][N][M]` → `T (*)[N][M]` |
| **二维数组退化** | `T [K][N]` → `T (*)[N]` |
| **类型匹配** | 退化后的指针类型必须与参数类型一致 |
| **常见错误** | 以为二维数组"套一层"就是三维数组 |
