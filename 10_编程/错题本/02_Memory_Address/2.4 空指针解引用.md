# 2.4 空指针解引用：NULL 不能读也不能写

## 🎯 题目 / 场景

向空指针写入数据导致程序崩溃：

```c
int *p = NULL;  // p指向地址0
*p = 100;       // ❌ 向地址0写入数据
```

---

## 💥 发生了什么

- **现象**：程序崩溃（Segmentation Fault / 段错误）
- **原因**：试图向地址0（NULL）写入数据
- **操作系统反应**：立即终止进程，防止破坏系统关键数据

---

## 🤔 我当时怎么想的

> "我先声明指针，等会再让它指向有效地址..."
> 
> **盲区**：不知道NULL（地址0）是操作系统保留区，**任何读写都会导致崩溃**。

---

## 🔍 根因分析

### 表面原因
> 指针未初始化或初始化为NULL，直接解引用写入

### 深层原理
> **地址0的特殊意义**：
> - 现代操作系统将地址0设为"禁区"
> - 用于捕获空指针解引用错误
> - 写入地址0会触发CPU的页保护机制
> - 操作系统收到信号后杀死进程（发送SIGSEGV信号）

### 识别信号
> **空指针解引用的3种常见场景**：
> 1. 指针声明后未初始化直接使用
> 2. malloc失败后返回NULL，未检查直接使用
> 3. 函数返回NULL表示失败，调用者未检查

---

## ✅ 正确规则与修正

```c
// ✅ 方法1：让指针指向已存在的变量
int a;
int *p = &a;   // p指向a
*p = 100;        // 安全，等价于 a = 100

// ✅ 方法2：动态分配内存
int *p = malloc(sizeof(int));
if (p != NULL) {  // ✅ 必须检查！
    *p = 100;
    free(p);
    p = NULL;     // 好习惯：防止悬空指针
}

// ✅ 方法3：如果可能为NULL，使用前检查
void process(int *p) {
    if (p == NULL) {
        printf("错误：空指针\n");
        return;
    }
    *p = 100;  // 现在安全了
}
```

## 🔁 一题多变 / 横向关联（视情况补充）

### 同类变式
- **读取空指针也会崩溃**
  ```c
  int *p = NULL;
  int x = *p;  // ❌ 同样是段错误
  ```

- **NULL检查的最佳位置**
  ```c
  // ❌ 不好：在函数内部深处检查
  void foo(int *p) {
      // ... 很多代码 ...
      if (p == NULL) return;  // 太晚了
  }
  
  // ✅ 好：在函数入口处检查（防御式编程）
  void foo(int *p) {
      if (p == NULL) return;
      // ... 后续代码可以安心使用p
  }
  ```

### 工程启示
> **防御式编程原则**：
> 1. 任何指针使用前检查是否为NULL
> 2. 函数参数如果是指针，入口处立即检查
> 3. malloc后**立即**检查返回值
> 4. 使用静态分析工具（如Clang Static Analyzer）自动检测空指针解引用
