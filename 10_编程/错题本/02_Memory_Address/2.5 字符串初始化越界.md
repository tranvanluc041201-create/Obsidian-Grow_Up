# 2.5 字符串初始化越界：别忘了 '\0'

## 🎯 题目 / 场景

字符串初始化时遗漏结束符：

```c
char c[5] = {'C','h','i','n','a'};  // ❌ 没有空间放'\0'
printf("%s", c);  // 越界读取，输出乱码
```

---

## 💥 发生了什么

- **预期**：输出 "China"
- **实际**：输出乱码（或者崩溃）
- **原因**：`printf("%s")` 会不断读取直到找到`'\0'`，但数组里没有

---

## 🤔 我当时怎么想的

> "5个字母就定义5个长度的数组，刚好..."
> "字符数组不就是字符串吗..."
> **盲区**：不知道C语言字符串必须以`'\0'`结尾，`printf`依赖这个哨兵。

---

## 🔍 根因分析

### 表面原因
> 数组长度没预留`'\0'`的空间

### 深层原理

**C语言字符串的本质**：
- 不是独立类型，只是**以`'\0'`结尾的字符数组**
- `printf("%s")`、`strlen`、`strcpy`等都依赖`'\0'`判断结束

**内存布局对比**：
```
// ❌ 错误
char c[5] = {'C','h','i','n','a'};
内存: [C][h][i][n][a]           (没有\0)

// ✅ 正确
char c[6] = "China";
内存: [C][h][i][n][a][\0]       (自动补\0)
```

### 识别信号
> **字符串长度 = 可见字符数 + 1（\0）**> 所有字符串操作函数都假设有`'\0'`，没有就是**未定义行为**。

---

## ✅ 正确规则与修正

```c
// ✅ 方法1：预留空间（推荐）
char c[6] = {'C','h','i','n','a', '\0'};  // 显式加\0
char c[6] = "China";                       // 自动补\0

// ✅ 方法2：让编译器自动计算
char c[] = "China";  // 编译器自动分配6字节

// ✅ 方法3：用字符串函数（安全）
char c[6];
strcpy(c, "China");  // 自动处理\0

// ❌ 危险：strcpy目标数组不够大
char c[3];
strcpy(c, "China");  // 缓冲区溢出！
```

## 🔁 一题多变 / 横向关联（视情况补充）

### 同类变式
- **strlen不算\0**
  ```c
  char s[] = "Hi";
  strlen(s);  // 返回2，不是3
  sizeof(s);  // 返回3（包含\0）
  ```

- **字符串数组**
  ```c
  char *words[] = {"Hello", "World"};  // 每个字符串自动有\0
  ```

### 工程启示
> **字符串安全3原则**：

> 1. 定义数组时长度 = strlen + 1
> 2. 使用strncpy代替strcpy（指定最大长度）
> 3. 始终确保字符串以`'\0'`结尾
>
> ```c
> // 安全版本
> char dest[10];> strncpy(dest, src, sizeof(dest) - 1);> dest[sizeof(dest) - 1] = '\0';  // 强制确保结尾
> ```
