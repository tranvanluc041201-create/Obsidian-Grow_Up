# 2.9 函数参数数组声明等价性

## 🎯 题目 / 场景

以下三个函数声明的关系是？

```c
void func(int a[3][4]);     // 声明1
void func(int a[][4]);      // 声明2  
void func(int (*a)[4]);     // 声明3
```

选项：
- A. 完全不同
- B. 前两个相同，第三个不同
- C. 完全相同
- D. 第一个不同，后两个相同

---

## 💥 发生了什么

**错误直觉**：以为 `int a[3][4]` 和 `int a[][4]` 一样，但 `int (*a)[4]` 是指针，应该不同。

**实际答案**：**C. 完全相同**

---

## 🤔 我当时怎么想的

> "数组和指针是不同的东西，虽然可以互相转换，但声明应该不一样..."
> 
> **盲区**：不知道在**函数参数**中，数组声明会**退化**为指针，而且第一维会被忽略。

---

## 🔍 根因分析

### 表面原因
> 不理解函数参数中数组声明的退化规则

### 深层原理

**C语言函数参数中的数组退化规则**：

```c
// 函数参数中的数组声明
void f(int a[3][4]);    // 实际等价于：void f(int (*a)[4])
void f(int a[][4]);     // 实际等价于：void f(int (*a)[4])
void f(int (*a)[4]);    // 本身就是指针形式
```

**为什么第一维会被忽略？**

```
内存布局理解：

a[0][0] a[0][1] a[0][2] a[0][3] | a[1][0] a[1][1] a[1][2] a[1][3] | ...
└───────── 第0行（4个int）────────┘└───────── 第1行（4个int）────────┘

编译器只需要知道：
1. 每行有多少个元素（第二维 = 4）→ 用于计算行偏移
2. 不需要知道有多少行（第一维）→ 不用于内存计算
```

**函数参数传递的本质**：
```c
// 调用时
int arr[3][4];
func(arr);  // arr退化为指向首元素的指针

// 首元素是"包含4个int的数组"
// 所以arr的类型是：int (*)[4]
```

### 识别信号
> **函数参数中的数组声明，第一维写不写都一样**
> ```c
> int a[N][M]  ≡  int a[][M]  ≡  int (*a)[M]
> // 只有第二维M是关键！
> ```

---

## ✅ 正确规则与修正

```c
// ✅ 这三种写法完全等价，任选其一
void func(int a[3][4]);     // 最直观，但第一维3会被忽略
void func(int a[][4]);      // 推荐！明确表示第一维可变
void func(int (*a)[4]);     // 最本质，显示指针本质

// 调用
int arr[3][4];
int arr2[5][4];
func(arr);   // ✅ 都可以
func(arr2);  // ✅ 都可以（只要第二维是4）
```

## 🔁 一题多变 / 横向关联（视情况补充）

### 同类变式

**一维数组的退化**：
```c
void f(int a[10]);   // 实际等价于 void f(int *a)
void f(int a[]);     // 实际等价于 void f(int *a)
void f(int *a);      // 本质

// 三种完全等价！
```

**三维数组的情况**：
```c
void f(int a[2][3][4]);     // 等价于 void f(int (*)[3][4])
void f(int a[][3][4]);      // 等价于 void f(int (*)[3][4])
void f(int (*a)[3][4]);     // 本质

// 只有第一维被忽略，第二、三维必须保留！
```

**容易混淆的对比**：
```c
// 函数参数中（退化发生）
void f(int a[3][4]);  // 等价于 int (*a)[4]

// 变量定义中（不退化）
int a[3][4];          // a是二维数组，不是指针！
int (*p)[4];          // p是指向数组的指针
```

## 🎯 关键记忆点

| 要点 | 口诀 |
|------|------|
| **函数参数数组** | 第一维可省，第二维必须对 |
| **数组退化** | `T a[N][M]` → `T (*a)[M]` |
| **本质理解** | 传的是指针，不是数组副本 |
| **等价声明** | `[N][M]` ≡ `[][M]` ≡ `(*)[M]` |
