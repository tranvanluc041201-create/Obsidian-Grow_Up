# Q8 结构体传参的“复印件”效应

## 🎯 题目 / 场景

```c
typedef struct {
    int x;
    int y;
} Point;

void move(Point p)
{
    p.x = 100;
}

int main(void)
{
    Point a = {1, 2};
    move(a);
    printf("%d %d\n", a.x, a.y);
}
```

---

## 💥 发生了什么

- 预期：调用 `move(a)` 后，`a.x` 变成 `100`
- 实际：输出仍然是 `1 2`
- 原因：函数参数 `p` 是结构体 `a` 的一份拷贝

---

## 🤔 我当时怎么想的

> “结构体这么大，传进去应该是共享同一份数据吧。”

---

## 🔍 根因分析

### 表面原因
- 把值传递误看成了“引用传递”

### 深层原因
- C 默认按值传参
- 对结构体来说，按值传参就是把整个结构体复制一份
- 在函数里改的是副本，不是原对象

### 识别信号
- 看到“函数里改结构体字段，外面为什么没变”时，先检查是不是按值传参

---

## ✅ 正确规则与修正

### 正确规则
- 想改原结构体：传指针
- 只想读结构体：按值或 `const` 指针都可以

### 最小修正代码
```c
void move(Point *p)
{
    p->x = 100;
}

move(&a);
```
