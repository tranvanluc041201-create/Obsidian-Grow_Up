# Q9 指针别名引发的数据覆盖

## 🎯 题目 / 场景

```c
int a = 3, b = 5, *p;
p = &a;

if (a < b) {
    *p = a;
    a = b;
    b = *p;
}
```

很多人会以为这是在交换 `a` 和 `b`。

---

## 💥 发生了什么

- 预期：`a` 和 `b` 交换，得到 `5, 3`
- 实际：最后可能得到 `5, 5`
- 原因：`*p` 不是临时仓库，而是 `a` 本身的别名

---

## 🤔 我当时怎么想的

> “先把 `a` 放进 `*p`，再改 `a`，最后再从 `*p` 里取回来，就能交换了。”

---

## 🔍 根因分析

### 表面原因
- 把“指针别名”误看成了“额外备份空间”

### 深层原因
- `p = &a` 后，`*p` 就是 `a`
- 所以 `*p = a` 实际上等于 `a = a`
- 当执行 `a = b` 后，`*p` 也跟着变成了新的 `a`
- 最后 `b = *p`，读出来的已经不是旧值 3，而是新的值 5

### 识别信号
- 看到 `p = &a` 之后，所有 `*p` 都应该立刻翻译成 `a`

---

## ✅ 正确规则与修正

### 正确规则
- 想交换两个变量，必须用真正的临时变量
- 指针指向原对象时，`*p` 不是副本，只是原对象的另一个名字

### 最小修正代码
```c
int t;
if (a < b) {
    t = a;
    a = b;
    b = t;
}
```

### 怎么自检
- [ ] 我能把 `*p` 全部翻译成它指向的对象
- [ ] 我能解释为什么 `*p = a` 没有真正“存住旧值”
