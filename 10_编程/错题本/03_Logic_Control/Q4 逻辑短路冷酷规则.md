# Q4 逻辑短路冷酷规则

## 🎯 题目 / 场景

逻辑短路导致++x未执行：

```c
int x = 0, y = 5, z = 3;
while (z-- > 0 && ++x < 5) {
    y--;
}
// 结果：x=3, z=-1
```

---

## 💥 发生了什么

**执行过程**：
```
第1次：z=3, 3>0(T), z变2, ++x=1, 1<5(T), 执行循环体
第2次：z=2, 2>0(T), z变1, ++x=2, 2<5(T), 执行循环体
第3次：z=1, 1>0(T), z变0, ++x=3, 3<5(T), 执行循环体
第4次：z=0, 0>0(F), z变-1, **短路！++x不执行**
```

**结果**：x=3（不是4），z=-1

---

## 🤔 我当时怎么想的

> "z--和++x都会执行..."
> "循环应该执行4次..."
> **盲区**：不知道&&左侧为假时，右侧会被短路跳过。

---

## 🔍 根因分析

### 关键点
- `z--` 是**后减**：先比较，再减1
- `++x` 是**前加**：先加1，再比较
- `&&` **短路**：左侧为假，右侧不执行

### 识别信号
> **循环条件中放++/--要小心短路**。

---

## ✅ 正确规则与修正

```c
// 如果希望++x每次都执行，调整逻辑
while (z-- > 0) {
    ++x;
    if (x >= 5) break;
    y--;
}
```
