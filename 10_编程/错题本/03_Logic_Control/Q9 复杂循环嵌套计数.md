# Q9 复杂循环嵌套计数

## 🎯 题目 / 场景

复杂循环嵌套计数：

```c
int a=10,b=5,c=5,d=5,i=0,j=0,k=0;
for (;a>b;++b) i++;        // 循环1
while (a>++c) j++;          // 循环2
do k++; while (a>d++);      // 循环3
printf("i=%d,j=%d,k=%d",i,j,k);
```

**答案**：i=5, j=4, k=6

---

## 💥 发生了什么

### 循环1：for
```
b从5跑到9：5,6,7,8,9（共5次）
i=5
```

### 循环2：while（++c前缀）
```
c=5→6, 10>6(T), j=1
c=6→7, 10>7(T), j=2
c=7→8, 10>8(T), j=3
c=8→9, 10>9(T), j=4
c=9→10, 10>10(F)停止
j=4
```

### 循环3：do-while（d++后缀）
```
先执行k++，再判断
...
d=10时：k++(k=6), 10>10(F), d变11停止
k=6
```

## 🤔 我当时怎么想的

> "++c和c++效果应该一样..."
> **盲区**：没仔细区分前缀/后缀自增的位置和时机。

---

## 🔍 根因分析

### 三种循环对比

| 循环 | 自增位置 | 执行次数 |
|------|---------|---------|
| for | ++b在条件后 | 5次 |
| while | ++c在条件内（前缀） | 4次 |
| do-while | d++在条件内（后缀） | 6次 |

### 关键点
- **前缀++**：先自增，再使用
- **后缀++**：先使用，再自增
- **do-while**：至少执行一次
