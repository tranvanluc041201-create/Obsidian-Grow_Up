# 4.1 Scanf 缓冲区残留：%d 后面的换行

## 🎯 题目 / 场景

连续使用scanf读取不同类型数据时出现问题：

```c
int num;
char ch;

printf("输入数字：");
scanf("%d", &num);      // 输入：42\n

printf("输入字符：");
scanf("%c", &ch);       // ❌ 没有等待输入，直接读取了\n
printf("数字=%d, 字符='%c'\n", num, ch);
// 输出：数字=42, 字符='
'（换行）
```

---

## 💥 发生了什么

- 预期：先读数字，再读字符
- 实际：字符读取直接跳过了，ch被赋值为回车符`\n`
- 原因：`scanf("%d")` 读取数字后，**回车符`\n`还留在输入缓冲区**

---

## 🤔 我当时怎么想的

> "scanf应该读完一个就清空缓冲区，下一个scanf从新的输入开始..."

> **盲区**：不知道scanf只读取匹配的部分，**空白字符（空格、回车）会留在缓冲区**。

---

## 🔍 根因分析

### 表面原因
> `%d`读取后，回车符`\n`残留在缓冲区，被下一个`%c`读取

### 深层原理
> **scanf的工作原理**：
> 1. `%d` 读取数字，遇到非数字字符（如`\n`）停止，但**不消耗**这个字符
> 2. `%c` 读取**任意字符**，包括空白字符
> 3. 所以`%c`读取了缓冲区里的`\n`

> **为什么`%s`没这个问题？** 因为`%s`自动跳过前导空白字符。

### 识别信号
> 凡是`scanf("%d")`后面跟`scanf("%c")`或`scanf("%s")`读单个字符/字符串的，都要注意缓冲区残留。

---

## ✅ 正确规则与修正

```c
// ✅ 方法1：用getchar()吃掉回车
scanf("%d", &num);
getchar();  // 吃掉\n
scanf("%c", &ch);

// ✅ 方法2：在%c前加空格（跳过所有空白字符）
scanf("%d", &num);
scanf(" %c", &ch);  // 注意%c前的空格！

// ✅ 方法3：用fflush(stdin)（不推荐，非标准C）
scanf("%d", &num);
fflush(stdin);  // 清空输入缓冲区（Windows可用，Linux可能不行）

// ✅ 方法4：用fgets+sscanf（最健壮）
char line[100];
fgets(line, sizeof(line), stdin);  // 读取整行
sscanf(line, "%d", &num);          // 解析数字
```

> 1. 简单场景：用`%c`前加空格的方法（最简单）
