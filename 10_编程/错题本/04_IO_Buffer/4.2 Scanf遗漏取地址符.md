# 4.2 Scanf遗漏取地址符

## 🎯 题目 / 场景

```c
int n;
scanf("%d", n);   /* ❌ */
scanf("%d", &n);  /* ✅ */
```

---

## 💥 发生了什么

- 预期：把输入的整数写进变量 `n`
- 实际：把 `n` 当前的值当成地址来写
- 结果：轻则数据错乱，重则直接崩溃

---

## 🤔 我当时怎么想的

> “`n` 就是我要输入进去的变量名，直接写进去就行。”

---

## 🔍 根因分析

### 表面原因
- `scanf` 需要的是“写入位置”，而不是变量当前值

### 深层原因
- `scanf` 本质上要把读到的数据写回内存
- 对普通变量来说，必须给它地址：`&n`
- 只有数组名这种本身能表示首地址的对象，才不需要再加 `&`

### 识别信号
- 看到 `scanf("%d" / "%f" / "%lf" / "%c", 变量)` 时，先问自己：这里传进去的是地址吗？

---

## ✅ 正确规则与修正

### 正确规则
- 普通变量给 `scanf` 时，默认都先检查有没有 `&`
- 只有字符数组配 `%s` 时，常见写法才是直接传数组名

### 最小修正代码
```c
int n;
double d;
char ch;
char s[100];

scanf("%d", &n);
scanf("%lf", &d);
scanf(" %c", &ch);
scanf("%99s", s);    /* s 不用再写 & */
```

## 🔁 一题多变 / 横向关联（视情况补充）

### 同类变式
- `scanf("%f", &x)` 里的 `x` 必须是 `float`
- `scanf("%lf", &x)` 里的 `x` 必须是 `double`
