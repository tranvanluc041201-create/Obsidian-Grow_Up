# 5.2 复合赋值隐式括号

## 🎯 题目 / 场景

```c
total *= sum + 12;
```

很多人第一眼会把它看成：

```c
total = total * sum + 12;   /* ❌ 错误理解 */
```

---

## 💥 发生了什么

- 预期：先乘再加
- 实际：右边会先当成一个整体
- 正确等价形式：`total = total * (sum + 12)`

---

## 🤔 我当时怎么想的

> “`*=` 就是把 `=` 前面的那部分补上去，后面还是按普通优先级展开。”

---

## 🔍 根因分析

### 表面原因
- 没把复合赋值看成一个“整体替换规则”

### 深层原因
- `a op= b` 的语义是 `a = a op b`
- 其中 `b` 是完整右侧表达式，不会被拆成“先跟 `a` 结合一半”

### 识别信号
- 看到 `+=`、`-=`、`*=`、`/=` 时，优先先展开成普通赋值再判断

---

## ✅ 正确规则与修正

### 正确规则
- 复合赋值先机械展开，再判断优先级

### 最小修正代码
```c
total *= sum + 12;
/* 等价于 */
total = total * (sum + 12);

count += i * 2;
/* 等价于 */
count = count + (i * 2);
```
