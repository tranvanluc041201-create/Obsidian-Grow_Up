# 5.3 八进制前缀陷阱

## 🎯 题目 / 场景

```c
printf("%d\n", 012);
```

很多人会条件反射写成 `12`。

---

## 💥 发生了什么

- 预期：输出 `12`
- 实际：输出 `10`
- 原因：以 `0` 开头的整数字面量，在 C 里按八进制解释

---

## 🤔 我当时怎么想的

> “前面的 `0` 只是写法问题，不影响值。”

---

## 🔍 根因分析

### 表面原因
- 把 `012` 当成普通十进制常量看了

### 深层原因
- C 里整数字面量有前缀规则：
  - `0` 开头：八进制
  - `0x` / `0X` 开头：十六进制
- 所以 `012 = 1 * 8 + 2 = 10`

### 识别信号
- 看到整数字面量前面有 `0`，先别急着按十进制读

---

## ✅ 正确规则与修正

### 正确规则
- 考试里看到 `012`、`010`、`077` 这类写法，优先先换算

### 最小修正代码
```c
printf("%d\n", 012);   /* 10 */
printf("%d\n", 10);    /* 10 */
printf("%x\n", 16);    /* 10 */
```
