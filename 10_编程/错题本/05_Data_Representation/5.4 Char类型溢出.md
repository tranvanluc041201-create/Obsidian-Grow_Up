# 5.4 Char类型溢出

## 🎯 题目 / 场景

```c
char c = 130;   /* 输出可能不是 130 */
printf("%d\n", c);
```

---

## 💥 发生了什么

- 预期：输出 `130`
- 实际：有的环境输出负数，有的表现依赖实现
- 原因：`char` 通常只有 1 个字节，取值范围有限

---

## 🤔 我当时怎么想的

> “`char` 也是整数类型，存个 130 应该没问题。”

---

## 🔍 根因分析

### 表面原因
- `char` 的取值范围容不下 `130`

### 深层原因
- `char` 是否有符号由实现决定
- 常见 `signed char` 范围是 `-128 ~ 127`
- 超范围赋值后会发生截断或实现相关结果

### 识别信号
- 看到 `char` 被拿来存数字，而数字大于 `127` 或小于 `-128` 时，要立刻警惕

---

## ✅ 正确规则与修正

### 正确规则
- `char` 主要用于存字符或很小的整数
- 存数值时，先确认范围是否安全

### 最小修正代码
```c
int x = 130;                 /* 最稳妥 */
unsigned char uc = 130;      /* 如果明确需要 0~255 */
printf("%d\n", x);
printf("%u\n", uc);
```
