# 5.5 浮点打印四舍五入

## 🎯 题目 / 场景

```c
float b = 9.8765f;
printf("%.2f\n", b);
```

---

## 💥 发生了什么

- 预期：很多人凭直觉会写成 `9.87`
- 实际：常见输出是 `9.88`
- 原因：`printf` 按显示精度进行舍入

---

## 🤔 我当时怎么想的

> “保留两位小数就是直接截掉后面的位数。”

---

## 🔍 根因分析

### 表面原因
- 把“格式化显示”误认为“直接截断”

### 深层原因
- `printf("%.2f", x)` 的含义是：按浮点格式输出，保留两位小数并做舍入
- 截断与舍入不是一回事

### 识别信号
- 看到 `%.nf` 这种写法，先想“显示精度”，不要先想“截断”

---

## ✅ 正确规则与修正

### 正确规则
- `printf("%.2f", x)`：显示时舍入
- 真要截断，得自己做数值处理

### 最小修正代码
```c
printf("%.2f\n", 9.8765);             /* 9.88 */
printf("%.2f\n", (int)(9.8765 * 100) / 100.0);  /* 9.87 */
```
