# 5.6 进制混淆与格式化输出

## 🎯 题目 / 场景

```c
unsigned int buffer[] = {0x10, 0x20, 0x30};
unsigned int *p = buffer;
printf("%d,0x%x\n", 012, *p++);
```

---

## 💥 发生了什么

- `012` 不是十进制 12，而是八进制 12，也就是十进制 10
- `*p++` 先取值，再移动指针，所以取到的是 `buffer[0]`
- `buffer[0]` 的值是 `0x10`，用 `%x` 打印时显示成 `10`
- 因此前半段是 `10`，后半段拼成 `0x10`

---

## 🤔 我当时怎么想的

> “前面的 012 就是 12，后面的 `%x` 只是多了个 0x 前缀。”

---

## 🔍 根因分析

### 表面原因
- 同时把“进制前缀”“格式控制符”“指针自增”都看错了

### 深层原因
- `012`：八进制常量
- `%x`：把数值按十六进制形式输出，不会自动加 `0x`
- `0x`：这里是格式串里的固定字符
- `*p++`：等价于 `*(p++)`，先解引用当前元素，再让指针后移

### 识别信号
- 题目里只要同时出现 `0` 前缀、`0x`、`%x/%o`、`*p++`，就必须拆开逐个算

---

## ✅ 正确规则与修正

### 正确规则
- 先换算字面量进制
- 再分清格式串里的字面字符和格式说明符
- 最后再单独分析指针表达式

### 最小修正代码
```c
unsigned int buffer[] = {0x10, 0x20, 0x30};
unsigned int *p = buffer;

printf("%d\n", 012);        /* 10 */
printf("%x\n", *p);         /* 10 */
printf("%d\n", *p++);       /* 16，再后移 */
```

### 怎么自检
- [ ] 我能说出 `012` 的十进制值
- [ ] 我能解释为什么 `%x` 输出的是 `10` 而不是 `0x10`
- [ ] 我能说出 `*p++` 为什么先取值后移动
