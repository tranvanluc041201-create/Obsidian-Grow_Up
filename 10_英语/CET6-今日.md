### 技术英语精读沉淀（今日）

- 材料：GitHub Trending / `promptfoo` README
- 类型：技术英语精读 + 中译英迁移

**原句 1**
- `promptfoo` is a CLI and library for evaluating and red-teaming LLM apps.
- 结构抓手：主系表；`for + V-ing` 放在表语内部，说明工具用途

**原句 2**
- Stop the trial-and-error approach - start shipping secure, reliable AI apps.
- 结构抓手：`stop A - start B` 表示停掉旧做法，换成新做法

**今日中译英**
- 中文：不要再靠反复试错来优化提示词，开始用可重复的评测来改进你的 AI 应用。
- 我的版本：Don't rely on trial-and-error approach to improve-Start using repeatable evaluations to improve your ai apps
- 更自然版本：Don't rely on trial and error to optimize your prompts. Start using repeatable evaluations to improve your AI apps.

**为什么这样改**
- `rely on` 是固定搭配，表示“依赖 / 靠”
- `trial and error` 常作整体方法说法，前面不必硬加 `approach`
- `start` 后面接动名词时，`start using` 比 `start use` 正确
- `repeatable evaluations` 比 `repeatable test` 更贴这类工具语境
- 同一小句里重复 `improve` 不算大问题；先求稳定自然，再谈变通

**今天带走的规则**
- 规则 1：工具介绍里常见 `for + V-ing`，多半是在说用途
- 规则 2：技术语境里 `shipping` 常表示交付 / 发布 / 真正上线使用
- 规则 3：中译英第一版先求搭配准，不要过早追求花样变通
