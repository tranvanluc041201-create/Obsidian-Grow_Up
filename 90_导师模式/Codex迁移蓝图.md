# Codex 迁移蓝图

> 目标：让 Codex 以五科导师模式为主入口，在分析与追问后，更稳定地生成并整理 Obsidian 学习笔记。

---

## 一、最终结论

这次迁移采用以下默认方案：

1. **五科都迁移**
2. **中文命名优先**
3. **Obsidian 继续做唯一真源**
4. **学科导师做主入口**
5. **学习整理助手做后置能力**
6. **学习中枢只负责兜底分流**

不再采用：

1. 把 Claude 规则全文硬拷到 Codex
2. 把模板、错题、笔记拆成多个公共 skill
3. 为了迁移而重命名现有中文目录

## 三、需求已经确认的部分

目前已经明确的用户需求有四条：

1. **五科统一管理**
2. **以中文命名为主**
3. **先进入学科导师模式**
4. **分析后追问学习重点，再调用模板做笔记**

这说明用户真正要的不是普通问答，而是这条统一工作流：

```text
进入学科导师
→ 分析题目/材料/视频
→ 追问本次笔记侧重点
→ 选择模板
→ 生成 Obsidian 笔记
→ 建议保存到对应学科目录
```

五科之间的差异主要体现在：

1. 学科规则不同
2. 模板不同
3. 目录落点不同
4. 追问重点不同

但工作流主干是一致的。

## 四、迁移原则

## 原则 1：学科导师是主入口

默认入口应是：

- `数学导师`
- `物理导师`
- `编程导师`
- `英语导师`
- `电气导师`

而不是让用户先思考：

- 这是模板问题吗
- 这是错题问题吗
- 这是笔记问题吗

因为用户的真实开口方式通常是：

- “开启数学导师模式”
- “英语导师模式，帮我分析听力”
- “编程导师模式，帮我看这段代码”

## 原则 2：整理能力是后置能力

“整理错题”“调用模板”“生成笔记”应视为同一段后置工作流。

所以公共能力只保留一个：

- `学习整理助手`

不再拆成多个 skill。

## 原则 3：Obsidian 是唯一真源

Codex skill 不直接承载全部规则，只负责：

1. 识别意图
2. 读取对应真源
3. 组织回答
4. 触发整理流程

知识与模板真源继续放在：

- `90_导师模式`
- `99_Templates`
- `10_数学` / `10_物理` / `10_编程` / `10_英语` / `10_电气`

## 原则 4：中文优先，路径不折腾

当前阶段不改：

1. 中文目录
2. 中文模板名
3. 中文学习体系命名

以实际可用和长期维护为先。

## 五、推荐结构

## 方案 B（最终确认版）

```text
学习中枢
学习整理助手
数学导师
物理导师
编程导师
英语导师
电气导师
```

### 角色分工

#### 学习中枢

作用：

1. 当用户没有明确说学科时，帮忙分流
2. 当用户只说“帮我整理”时，判断是直接整理还是先进入学科导师

#### 学习整理助手

作用：

1. 接收学科导师的整理需求
2. 追问笔记侧重点
3. 选择模板
4. 输出适合 Obsidian 的 Markdown
5. 建议保存路径

它不是默认主入口，而是学科导师常用的后置助手。

#### 五个学科导师

作用：

1. 陪学
2. 分析问题
3. 使用各自 L1/L2/L3 规则
4. 判断何时进入整理流程

## 七、最小可用闭环

当前最值得先做通的，不是七个 skill 全量完成，而是一个最小可用闭环。

### 第一批闭环

1. `数学导师`
2. `学习整理助手`
3. `英语导师`

### 原因

1. 数学已经有明确高频场景
2. 学习整理助手是五科共用能力
3. 英语最特殊，越早处理越能避免后面返工

---

## 八、导师与整理助手的协作关系

默认协作顺序如下：

```text
用户进入某学科导师
→ 导师分析当前问题或材料
→ 导师追问本次目标与笔记侧重点
→ 需要整理时调用学习整理助手
→ 整理助手选模板并生成 Markdown
→ 导师给出保存建议
```

重点是：

1. **先分析，再追问**
2. **先明确笔记侧重点，再选模板**
3. **不要一上来就生成笔记**

## 九、验证标准

## 1. 陪学验证

示例：

- “开启数学导师模式，帮我整理视频学习笔记”
- “英语导师模式，帮我分析这段听力”
- “编程导师模式，帮我看为什么报错”

验证点：

1. 是否进入正确学科
2. 是否先分析材料
3. 是否主动追问本次侧重点
4. 是否按学科规则组织回答

## 2. 笔记验证

示例：

- “把这道极限题整理成数学错题笔记”
- “把这段听力分析整理成英语错题卡”
- “把这次电路分析整理成电气项目笔记”

验证点：

1. 模板是否贴合学科
2. Markdown 是否适合 Obsidian
3. 保存路径建议是否正确
4. 标题与结构是否中文优先
