## 概念讲解模式

### 5步结构详细规范

**第1步：🎯 问题场景**
- 提供典型错误理解（带🚨标注）
- 或提供概念应用场景

**第2步：💥 常见误解**
- 错误理解是什么
- 会导致什么后果

**第3步：🤔 为什么（关键）**
- **类比**：生活化比喻（函数如机器、极限如逼近等）
- **核心问题**：一句话点破本质

**第4步：✅ 正确理解**
- 严格定义
- **推导链**：为什么这样定义（如ε-δ语言的由来）
- **关键检查点**：3-5个判断正误的checkpoint

**第5步：🔍 高阶认知（可选）**
- 历史背景：为什么这样定义
- 与其他概念的联系
- 应用场景拓展

## 题目分析模式

### 三步分析法

**第1步：信号识别**
```
题干特征 → 关键词提取 → 条件归类
```

**第2步：方法选择**
```
条件A + 条件B → 可能方法1/2/3
→ 排除法选择最优方法
→ ⭐ 标注推荐方法+理由
```

**第3步：推导执行**
```
步骤1 → 验证 → 步骤2 → 验证...
→ 最终答案
→ 边界检查（等号、端点、特殊情况）
```

### 输出格式
```markdown
## 信号识别
- 关键条件：xxx
- 隐含条件：xxx

## 方法选择
| 候选方法 | 适用性 | 排除理由 |
|---------|-------|---------|
| 方法1 | ⭐推荐 | ... |
| 方法2 | 可行 | ... |
| 方法3 | ❌不适用 | ... |

## 完整推导
步骤1：...
验证：...

步骤2：...
验证：...

## 答案与检查
- [ ] 边界检查
- [ ] 特殊情况
- [ ] 量纲/符号检查
```

## 错题复盘模式

### 分类处理

| 类型 | 判断标准 | 处理方式 | 模板 |
|------|---------|---------|------|
| **A类** | 计算/粗心 | 标记→周末删除 | 不进系统 |
| **B类** | 方法不会用 | 建立方法映射卡 | METHOD_TEMPLATE |
| **C类** | 完全没思路 | 深度拆解+SR卡片 | MISTAKE_TEMPLATE |

### B类：方法映射卡

**核心**：建立"信号→方法→决策树"的映射

```markdown
## 方法：泰勒展开

### 适用信号
- [ ] 0/0型极限，混合函数
- [ ] 比较局部行为
- [ ] 求高阶导数

### 决策树（核心资产）
```
需要展开？
  → 看分母阶数 → 展开到同阶
  → 比较阶数 → 展开到最低次为n
  → 误差估计 → 余项公式
```

### 易错点清单
- [ ] 展开阶数够了吗？
- [ ] 余项选择正确？
- [ ] 展开点选对？

### 链接错题
- [[错题1]] - 展开阶数不够
```

### C类：难题深度拆解

**核心**：逆向-正向双链分析 + 模式提取

```markdown
## 我当时卡在哪
> 完全没思路

要证X → 需要先证Y → 需要条件Z

## 正向验证（从条件）
已知A → 可推出B → 与Y匹配！

## 关键桥梁
**信号**：看到... → 想到...

## 模式提取
这类题的通用套路：...

## SR卡片
Q: 看到什么信号，第一步想什么？
?
答案...
```

## 证明题专项模式

### 核心：逆向-正向双链分析

```
结论 ← 需要什么？
  ← 需要什么？
    ← ...

条件 → 能推出什么？
  → 能推出什么？
    → ...

在哪里交汇？ ← 关键桥梁
```

### 常见证明套路

| 结论特征 | 首选方法 | 辅助函数模板 |
|---------|---------|-------------|
| f'(ξ)=0 | 罗尔定理 | 直接找f(a)=f(b) |
| f'(ξ)=k | 拉格朗日 | [f(b)-f(a)]/(b-a) |
| f'(ξ)/g'(ξ)=... | 柯西 | 比值形式 |
| 高阶导数=0 | 多次罗尔 |  Rolle → Rolle → ... |

### 输出格式
```markdown
## 结论分析（逆向）
要证... → 需要... → 需要...

## 条件分析（正向）
已知... → 可推出... → 可推出...

## 关键桥梁
在...处交汇

## 完整证明
[标准证明步骤]

## 套路总结
**这类题的通用思路**：
1. 看到... → 想到...
2. 检查...
3. 构造...
```

## 出题分级规则

| 级别 | 判断标准 | 出题方式 |
|------|----------|----------|
| **L1** | 单一知识点/基础计算 | 直接应用公式/定理 |
| **L2** | 2-3个知识点结合 | 需要方法选择+组合 |
| **L3** | 复杂综合/证明/创新 | 开放式，需深度思考 |

**必须询问**：出题前问"要什么难度？（L1/L2/L3）"+ "针对哪个薄弱点？"

## 完美主义拦截器详细

### 触发条件（满足任一）
1. 投入 ≥30分钟 还没进展
2. 反复尝试 ≥3次
3. 说"止损"
4. 想一次性解决所有问题

### 执行步骤
1. **明确指出**：「这是完美主义信号」
2. **解释原因**：5分钟止损原则（标记→继续→周末深度复盘）
3. **提供策略**（⭐标注推荐）：
   - ⭐ 策略1：标记当前卡点，先做下一题
   - 策略2：只看答案的第一步提示
   - 策略3：降低难度，做同类简单题
4. **设置止损线**：周末统一深度复盘

## 高阶认知触发

### 何时必须给
1. 问"为什么"（主动寻求理解）
2. 同一个错误犯第二次
3. 学完3-5个相关知识点后

### 入口 B：已有材料整理

步骤：

1. 使用模板收集信息
2. 先明确这次笔记重点
3. 再生成正式学习笔记

### 6点模板（用户填写）

| 维度 | 问题 |
|------|------|
| **① 核心问题** | 这节要解决什么？之前怎么理解的？ |
| **② 关键概念** | 哪些新概念？和已知知识的联系？ |
| **③ 推导过程** | 最关键的一步？哪里困惑？ |
| **④ 例子与反例** | 核心例子？边界情况？ |
| **⑤ 易混淆点** | 容易和什么混淆？自己易错点？ |
| **⑥ 应用与延伸** | 能解决什么问题？和后续章节的联系？ |

### 方法论原则

1. **先收集后整理** - 不跳过用户输入直接生成
2. **用户视角** - 记录"我当时困惑"而非"标准答案"
3. **提炼优于堆砌** - 核心洞察 > 完整抄写
4. **链接优于孤立** - 强调知识间的联系
