# 物理视频学习流程模板

> 用途：当用户说“物理视频学习流”时，默认按“学完视频后复盘”的 `4+1 闭环` 处理。
> 说明：这是一张执行卡，不是长期规则真源。长期规则以 `[[10_物理/快速开始]]` 和 `[[90_导师模式/Physics/01_L1_CORE]]` 为准。

## 零、适用范围

- 力学、电磁学、热学等物理课程视频
- 用户已经看完或基本看完一节课
- 目标是复盘、图像重构、Pattern 提取和沉淀

默认不把这张模板当成“边看边停的陪看流”。
也不要把它做成“用户只丢链接，AI 直接代写成品笔记”的流程。

---

## 一、默认流程：4+1 闭环

### 1. 课后回忆

先收用户自己的学习痕迹，只要最小 3 项：

1. 这节讲了什么对象或现象
2. 主线是什么
3. 最不稳的一点是什么

可选补充：

- 视频主题或链接
- 3 到 5 个关键词
- 一张板书截图或一小段提纲

如果用户当前没有给出这 3 项，第一次回复先追问这 3 项，不直接产正式笔记。

原因：

- 对“看完就忘”型用户，先让用户闭卷输出，比先看成品笔记更有效
- 这一步的作用是强制检索，不是礼貌性提问

---

### 2. 图像重构

固定只重构 4 个东西：

1. **对象 / 场景**
   - 研究对象、过程或系统是什么
2. **微图解**
   - 受力图 / 场线图 / 回路图 / 过程图是什么
3. **条件 / 近似**
   - 哪些条件成立，哪些近似在用
4. **关键关系**
   - 守恒、定律、趋势或核心关系是什么

如果重构开始发散，就退回这 4 项。

---

### 教材按需插入

默认顺序是：`先视频 -> 先回忆/图像重构 -> 再决定要不要查教材`。

只在下面三种情况读教材：

1. `预热`
   仅当这节新模型、新场景、新符号较多，或先修概念明显不足时。
   默认只看 `3~5` 分钟：对象定义、图示、条件与近似。
2. `纠偏`
   图像重构后，如果对象、近似条件、适用范围或标准推导仍不稳，再查教材对应 `1~3` 页。
3. `定稿`
   只有在正式笔记前，才用教材统一图像、条件表述和标准公式。

不要做：

- 不要把整章预读变成默认步骤
- 不要在第一次图像重构前先翻教材找标准答案

---

### 3. 单点验证

四选一，默认只做一个：

1. 一道最小例题
2. 一个判断题或口头判断
3. 一次数量级 / 极限 / 量纲检查
4. 和旧题或旧错题做一处映射

目标不是做很多题，而是验证是不是“真懂了”。

---

### 4. 单卡沉淀

五选一，默认只产出一张卡：

1. **学习卡**
   - 主线 + 图像 + 关键关系
2. **Pattern 卡**
   - 识别条件 + 解题顺序
3. **错因卡**
   - 本节暴露出的错因与检查点
4. **概念卡**
   - 一个最核心概念
5. **检查卡**
   - 量纲 / 极限 / 数量级的检查提醒

不要默认写成长笔记。

---

### 5. 次日回看

只做 2 分钟闭卷回忆：

1. 这节的图像是什么
2. 最核心的关系是什么
3. 上次最不稳的一点现在会不会了

这一步高于“重看整节视频”。

---

## 二、第一次回复必须给出

1. 当前输入里已经有什么
2. 用户自己的回忆里，哪一点是对的，哪一点是模糊或错的
3. 你判定这次更适合哪一种卡片
4. 当前最推荐的单点验证是什么
5. 现在最该做的下一步是什么

不要一上来就写大总结。
如果用户没有先输出自己的回忆，也不要一上来直接写正式笔记。

---

## 三、什么时候升级到重流

只有满足下面任一条件，才升级：

1. 用户明确说这节是核心难点
2. 回忆结果明显发散
3. 板书、分类讨论或图像信息特别密
4. 用户明确说要正式笔记

升级后，额外只补下面 2 项：

1. 更完整的 Pattern 或错因分析
2. 正式笔记骨架

不要默认每次都升级。

---

## 四、正式笔记前的动态筛问

只有当用户明确要整理成正式笔记时，才开启这一轮筛问。

不要固定把问题问死。

先保留最小固定输入：

1. 这节讲了什么对象或现象
2. 你现在认为的主线是什么
3. 你最不稳的一点是什么

然后再根据视频内容，从下面题库里只选 `2~4` 个最值得问的问题。

筛选依据：

1. 这节更偏概念、Pattern、错因，还是复习速查
2. 用户的回忆暴露了哪类薄弱点
3. 这篇笔记是为了次日回忆、做题前速查，还是错题返修

### 动态题库

#### 图像 / 概念型

- 这节最该保留的图像到底是哪一张
- 哪个条件或近似一旦漏掉，整节就会理解错
- 你更需要记“概念解释”还是“图像判断口令”

#### Pattern / 做题顺序型

- 这节最该记住的识别信号是什么
- 你最容易在哪一步选错原理或顺序
- 这篇笔记更该保留“步骤顺序”还是“分叉判断点”

#### 错因 / 检查型

- 你现在最容易错在图像、条件、原理，还是检查
- 这节最值得沉淀成哪一种检查点：量纲 / 极限 / 数量级
- 这篇笔记要不要把一个真实错因作为主线

#### 复习速查型

- 你以后最可能在什么场景下翻这篇：次日回看 / 一周复习 / 做题前
- 你需要的是极简回忆卡，还是标准正式笔记

### 默认映射

- 图像 / 概念 / 复习速查 → `[[99_Templates/物理/视频笔记模板|物理视频笔记模板]]`
- 错因 / 检查 → `[[99_Templates/物理/错题模板|错题模板]]`
- Pattern / 做题顺序 → `[[99_Templates/物理/练习模板|练习模板]]`

## 五、与电气的衔接

默认仍然归 `物理导师`，但不要和 `电气导师` 完全隔绝。

当视频内容明显进入电路化表达时，可以借用电气导师的一个收束方式：

1. 对象是什么
2. 关键关系是什么
3. 应该检查什么

如果已经进入电路分析、控制、仿真、PCB/EDA，就切到 `电气导师`。

## 六、推荐路径

| 产物 | 路径 |
|------|------|
| 当天复盘记录 | `10_物理/每日记录/` |
| 概念卡 | `10_物理/01_力学/概念/` 或对应学科概念目录 |
| Pattern / 练习 | `10_物理/练习_临时/` |
| 错因或错题 | `10_物理/错题_临时/` 或对应学科错题目录 |

第一次跑流程，默认先落到每日记录或临时目录。

---

## 七、验收标准

- [ ] 用户自己的回忆已收集
- [ ] 4 项图像重构已完成
- [ ] 已做一个单点验证
- [ ] 已产出一张卡
- [ ] 已给出次日回看点
- [ ] 如果要写正式笔记，已先完成问答定侧重点

少任意一项，都不算完整跑完。

---

## 八、止损

- 回忆太散时，只抓一个主问题
- 内容太多时，不补第二张卡
- 例题太重时，退回判断题或检查题
- 笔记开始变长时，立刻收回到单卡
- 物理图像不清时，先停在图像重构，不急着算

---

## 九、何时切回别的流程

- 具体题目详解：切回 `物理导师`
- 整理成正式 Obsidian 笔记：再衔接 `学习整理助手`
- 设计或修改整套学习流：先回 `learning-flow`
