## 4. 这张图不要出现什么

- 不要大段解释
- 不要一张图塞完整本书
- 不要把导航和关系混在一起
- 不要默认靠点击展开

补充规则：

- 已会：默认只留坑点
- 会概念但写不出：默认补“代码桥”
- 未学：默认补“定义 + 最小写法 + 高频坑”

## 6. 命名规则

- 偏短词 / 偏提示词 / 偏术语：
- 是否接受符号节点（如 `a`、`&a`、`sizeof`）：
- 是否需要最小例子：
- 是否需要“代码桥节点”（参数顺序 / 最小模板 / 写法对照）：
- 是否需要单独补一张“跨图串联图”：

## 7. 协作检查点

- 第一轮：先判“会 / 似懂非懂 / 不会”
- 第二轮：如果用户不确定知识点，就先看候选版
- 第三轮：只看哪些点要加深
- 第四轮：只删废话，不乱删分支

## 8. 最终验收

- [ ] 一眼能看出主干
- [ ] 高频坑直接显示
- [ ] 节点够短
- [ ] 不靠点击才能理解
- [ ] 没把“会表面但有坑”的点误删
- [ ] “会概念但写不出”的点已经补了代码桥
- [ ] 跨图可类比的点已经串起来
