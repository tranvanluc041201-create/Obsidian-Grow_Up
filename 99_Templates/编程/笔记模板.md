## 示例：malloc/free 笔记

```markdown
# 5）malloc/free

## 🎯 一句话结论
malloc 向操作系统申请内存，用完必须 free，否则内存泄漏。

---

## 💥 问题场景

**错误的代码**：
```c
int *a = malloc(3 * sizeof(int));
a[0] = 1;
// 没 free 就结束
```

**会发生什么**：
程序结束后，那块内存还被标记为"在用"，操作系统收不回来（内存泄漏）。

---

## 🤔 为什么（通俗理解）

**类比**：
malloc = 向图书馆申请一个储物柜
free = 用完归还储物柜

没 free = 你走了但柜子还占着，别人用不了。

**核心问题**：
C 不像 Python 有垃圾回收，必须手动管理内存生命周期。

---

## ✅ 正确写法

```c
int *a = malloc(3 * sizeof(int));
if (a == NULL) {
    return 1;  // 申请失败处理
}

// 使用...
a[0] = 1;

free(a);
a = NULL;  // 好习惯：防止后续误用
```

**关键检查点**：
- [ ] 是否检查了 malloc 返回值（NULL）？
- [ ] 每个 malloc 都有对应的 free？
- [ ] free 后是否置 NULL（可选但推荐）？

---

## 🔍 高阶视角

**工程思维**：
为什么 C 要手动管理？因为自动回收需要额外开销（性能损耗）。
C 选择：给你最大控制权，你自己负责。

**类似场景**：
- 文件 fopen → fclose
- 网络 socket → close
- 锁 lock → unlock

资源获取必须配对释放，这是系统编程的铁律。
