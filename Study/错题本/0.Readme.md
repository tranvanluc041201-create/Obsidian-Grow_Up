1.[📁 01_Compile_Link (编译/语法)](./01_Compile_Link/)
定义：代码还没跑起来就报错，或者生成可执行文件失败。
收录内容：

    语法错误：少分号，括号不匹配。

    类型冲突：把 float 强行赋给 int 指针。

    链接错误：函数声明了但未定义 (undefined reference)，头文件重复包含 (#ifndef 缺失)。

2.[📁 02_Memory_Address (内存/指针)](./02_Memory_Address/)
定义：涉及指针、数组、栈、堆的所有问题。这是 C 语言的“高压危险区”。
收录内容：

    指针错误：野指针、空指针解引用。

    数组越界：a[10] 访问了 a[11]。

    内存模型：多维数组的线性存储步长问题。

    内存泄漏：malloc 后没有 free。

3.[📁 03_Logic_Control (逻辑/算法)](./03_Logic_Control/)
定义：程序能跑，也没有内存错误，但结果不对（算错了、死循环了）
收录内容：

    循环陷阱：for 循环边界搞错，死循环。

    分支逻辑：switch 误用布尔值，if (a=b) 赋值当判断。

    算法缺陷：递归没写终止条件，排序算法写错。

4.[📁 04_IO_Buffer (输入/输出)](./04_IO_Buffer/)
定义：程序和外部世界交互时的问题。
收录内容：

    缓冲区残留：scanf 遗留回车符导致跳过输入。

    格式化字符串：printf("%d", float) 导致的乱码。

    文件操作：文件打不开，读取 EOF 判断错误。

5.[📁 05_Data_Representation (数据/溢出)](./05_Data_Representation/)
定义：计算机底层如何存储数据的问题。
收录内容：

    浮点精度：0.1 + 0.2 != 0.3。

    整数溢出：short 类型存了 32768。

    ASCII 陷阱：字符 '0' 和数字 0 混淆。
