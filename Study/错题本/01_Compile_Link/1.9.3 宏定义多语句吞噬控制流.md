### [1.9.3] 宏定义多语句吞噬控制流 (Swallowed Scope)
* **标签**：#Macro #Preprocessor

**典型错误**：
```c
#define RESET_PERIPHERAL() \
    GPIO_A = 0; \
    GPIO_B = 0;
hardware_error=0;
知识点:"\"为转行符,表示:将下一行和上一行看作同一行
因此等价于:#define RESET_PERIPHERAL() GPIO_A = 0; GPIO_B = 0;------不要被;误导
                                                                宏定义不需要;表示定义结此时;被看作里面的一部分内容
// 调用
if (hardware_error)
    RESET_PERIPHERAL();
```
**错误直觉**：认为只有在 error 时才复位两个 GPIO。
**实际结果**：GPIO_A 随 if 执行，而 GPIO_B 永远被置 0。
**文本展开**：
```c
if (hardware_error)
    GPIO_A = 0;
GPIO_B = 0; // 它是孤儿，不属于 if
``
**修正代码**：
```c
#define RESET_PERIPHERAL() do { \
    GPIO_A = 0; \
    GPIO_B = 0; \
} while (0)
```
**规范**：《OpenHarmony 规范》规则 6.2：包含多条语句的函数式宏必须放在 do-while(0) 中。
