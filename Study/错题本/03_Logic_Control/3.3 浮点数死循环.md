### [3.3] 浮点数死循环
* **标签**：#Float #Loop
* **错误**：`for(float i=0; i!=1.0; i+=0.1)`
* **根因**：**精度丢失**。
关键点：0.1 在二进制浮点里通常表示不精确（像 1/3 在十进制里写不尽一样）。

所以 i += 0.1 实际是在加一个“接近 0.1 的数”，误差会累积，导致：

i 可能变成 0.99999994、1.0000001 之类

很可能永远不会刚好等于 1.0

那么 i != 1.0 就一直为真 → 循环不停 ✅
* **修正**：用整数计数 `int i`
* #include <stdio.h>

int main(void) {
    for (int k = 0; k <= 10; k++) {
        float i = k / 10.0f;   // 关键：10.0f 让除法变成浮点除法
        printf("k = %2d, i = %.7f\n", k, i);
    }
    return 0;
}
