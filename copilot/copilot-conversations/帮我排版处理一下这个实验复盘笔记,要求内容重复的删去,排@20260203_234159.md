## 📊 系统工程讲解

### 1. SITO 能量路径（不是"三个 Buck 叠一起"）

**核心本质**：一个电感电流 $I_L$ 被时间复用，按时隙把能量"分发"给不同输出。

#### 三个动作模块：

**① 充能（Charge）**：主开关导通，电感电流上升
- 直觉：电感像"能量背包"，ON 时往背包里塞能量。
- 路径：$V_{in} \to$ 主开关 S1 $\to$ L（充能）$\to$ 被选通的输出支路

**② 分发（Allocate）**：电感放能，但"吐给谁"由支路选通决定
- SITO 的核心不是三路 PI，而是"分发阀门"：同一股 $I_L$ 分时喂给 V1/V2/V3
- 分发失败通常不是控制坏了，而是：**选通器件/回路/参考点让能量根本没按预期路走**

**③ 续流（Freewheel）**：主开关关断后，电感电流必须有路走
- 路径：L（放能）$\to$ 续流路径（二极管/同步开关）$\to$ 被选通的输出支路
- **排错口令**："电感电流现在走哪条路？" —— 只要能回答这句，-600k 这种离谱值就很难困住你

#### 为什么能直接当排错工具
功率电路里最真实的不是电压，是**电流路径**：
- 电压离谱时，先问：电感电流到底流到哪儿去了？有没有通路？有没有被错误回流？
- MOSFET 接反/体二极管方向错，本质就是"以为有路"，实际是**把路堵死或引到反方向**

---

### 2. 控制链路解释（用类比讲清楚）

#### 链路结构
```
Vref_i ─┐
        ├─(误差 e_i = Vref_i − Vfb_i) → PI_i → Limiter(0..1) → 与 ramp 比较 → PWM_i → 控制开关
Vfb_i ──┘
```

#### PWM 的本质（不是"魔法波形"）
- **Comparator vs ramp** 本质上是在做"定时"：
  - ramp 是一个每周期从 0→1 的"计时尺"（50kHz）
  - Limiter 输出是你想要的"时间比例"（即占空比 $d$）
  - 比较器说：`当 ramp < d 时，PWM=1；否则 PWM=0`
  
- 所以只要 **ramp 与 d 同量纲**（0..1 对 0..1），PWM 的占空比就自然等于 $d$

#### 开环 vs 闭环（强力类比）
- **开环**：你固定给一个 $d$（比如 0.3），不管输出漂到哪。像"按经验每天吃同样剂量的药"。
- **闭环**：你测 Vfb，和 Vref 比，算出误差 $e$，再改 $d$。像"测血糖→算差→调胰岛素剂量"。

#### 这条链路里"最容易坏在哪里"（按坏的概率）
| 模块 | 问题 | 表现 | 优先级 |
|---|---|---|---|
| PI | 饱和、积分 wind-up | 长期顶死 0/1 | ⭐⭐ |
| **Limiter** | **量纲错** | **比较器直接失明** | **⭐⭐⭐⭐⭐** |
| **Comparator** | **极性反** | **控制方向反着跑** | **⭐⭐⭐⭐** |
| **PWM→Switch** | **MOSFET 浮地 Vgs 不成立** | **开关永远没真打开** | **⭐⭐⭐⭐⭐** |

### 三分钟排错剧本（3-step workflow）

#### Step 0（30s）：打开强制观测窗（同屏）
必看信号：
- `Vout1/2/3`, `Vfb1/2/3`, `e1/2/3`, `PIout`, `Limout`, `ramp`, `PWM1/2/3`, `IL`

**判据**：如果看不到 `d vs ramp` 与 `PWM` 的对应关系，就别改电路，先把观测补齐。

#### Step 1（60s）：看 Limout 是否卡死
1. 检查 `Limout` 是否在 0..1 **且不长期贴边**
2. 检查 `ramp` 是否 0..1 且频率正确（50kHz）
3. 检查 `Comparator` 极性：`ramp < d → PWM=1`（用固定 d=0.3 验证最直观）

**如果卡死**：先查误差极性、Limiter 范围、ramp 范围一致性

#### Step 2（60s）：做"参考点硬拉"验证
1. 把关键传感器负端直接拉到 VDC− 黑点节点
2. 观察输出是否立刻恢复

**如果立刻正常**：根因 = 参考点不一致，统一所有地后 END

**如果仍异常**：进入 Step 3

#### Step 3（30s）：确认 Vgs/开关真导通
1. 直接测 $V_{gs} = V_{gate} - V_{source}$ 是否足够
2. 或改用 Controlled/Ideal Switch 做对照

**如果 Vgs 太小**：根因 = 浮地驱动，改模型或做高边驱动参考

### 隔离法（秒判"控制问题 vs 功率问题"）

#### 隔离法 A：固定占空比替代 PI
- 断开 PI 输出 → 用 DC = 0.3/0.5 直接喂 Comparator
- 若输出能被拉起来 → 控制链大体没死，回头调 PI
- 若仍拉不起来 → 优先查开关模型/能量路径/参考点

#### 隔离法 B：单路跑通再叠加
- 只保留 V1（或只保留 V2）支路，把另外两路选通关死
- 能单路稳 → 功率级基本合理
- 单路都不稳 → 先把基础 Buck 跑稳，再谈三路耦合

## ❓ 常见问题与器件选型

### Q1：PSIM 里"红线/绿线"是什么意思？

**A**：两类网络分别承载不同信号类型：
- **Power 网络（红线）**：承载功率电流/电压的电气连接
- **Control/Logic 网络（绿线）**：承载控制信号（0/1、PWM、比较器输出）

你说的"从 on-off switch 出来就是红色，改不了"，说明：**你选的 on-off switch 是"功率器件"而不是"逻辑控制元件"**，它输出不是绿线，所以没法当 PWM 控制信号。

---

### Q2：On-Off Switch vs Bi-directional Switch 的区别？

| 器件 | 用途 | 控制方式 | 适用场景 |
|---|---|---|---|
| **On-Off Switch** | 开/关某条功率支路或测试隔离 | 不一定有控制端口（绿线） | 手动/固定隔离，**不适合 PWM 控制** |
| **Bi-directional Switch** | 功率级开关器件，可双向导通 | 需配套 Switch Controller 受控 | **支路选通/背靠背 MOSFET 等效** |

---

### Q3：你现在"支路选通/背靠背"场景应该用哪个？

**答案**：用 **Bi-directional Switch**（功率开关）+ **Switch Controller**（控制器）

**理由**：你需要"受 PWM 控制的功率路径选通"，而不是"手动开关"。

**PSIM 里的实际操作**：
1. 功率器件：**Power → Switches → Bi-directional Switch**（你截图里就在列表）
2. 控制器件：同一 Library Browser 左侧 → **Switch Controllers**（你截图里有文件夹名）
3. 用你的 PWM/逻辑信号（绿线）去驱动控制器，再由控制器去驱动 Bi-directional Switch

> 💡 提示："找不到 Controlled Switch (logic)" 是正常的 —— 你的库里就没这个名字。别硬搜字符串了，按上面两段式（Switch + Controller）走。

---

### Q4：为什么 MOSFET 浮地驱动这么要命？

**A**：MOSFET 导通取决于 $V_{gs} = V_{gate} - V_{source}$，不是"Gate 对地电压"。

- Comparator 输出是"对地的电压/逻辑"
- 但若 Source 在漂移点（比如在变压的节点上），$V_{gs}$ 可能很小
- 结果：**波形看起来在 PWM，器件实际没真正导通**

**快速检查**：直接测 $V_{gs}$ 的值；或改用 Controlled/Ideal Switch 完全避免这个问题。

## 📇 可复用模块库（下次搭建时直接拿）

### 模块 1：Error + PI + Limiter（每路一个）
- **输入**：$V_{ref,i}$, $V_{fb,i}$
- **输出**：$d_i$（0..1）
- **判据**：稳态时 $d_i$ 在 0..1 中间某个值，不应长期卡死边界

### 模块 2：Ramp + Comparator → PWM
- **输入**：$d_i$（0..1）、ramp（0..1）
- **输出**：$PWM_i$
- **判据**：$d_i$ 越大，PWM 高电平占比越大；量纲必须一致

### 模块 3：选通/开关驱动
- **输入**：$PWM_i$
- **输出**：支路导通状态
- **注意**：若用 MOSFET，高边/浮地必须确保 Vgs；否则用 Controlled/Ideal Switch 更省心

## 📚 系统工程讲解

### 1. SITO 能量路径（核心本质）

**一句话模型**：一个电感电流 $I_L$ 被时间复用，按时隙把能量"分发"给不同输出。

#### 三个动作模块

**① 充能（Charge）**：主开关导通，电感电流上升
- 直觉：电感像"能量背包"，ON 时往背包里塞能量
- 路径：$V_{in} \to$ 主开关 S1 $\to$ L（充能）$\to$ 被选通的输出支路

**② 分发（Allocate）**：电感放能，但"吐给谁"由支路选通决定
- SITO 的核心不是三路 PI，而是"分发阀门"：同一股 $I_L$ 分时喂给 V1/V2/V3
- 分发失败通常不是控制坏了，而是：**选通器件/回路/参考点让能量根本没按预期路走**

**③ 续流（Freewheel）**：主开关关断后，电感电流必须有路走
- 路径：L（放能）$\to$ 续流路径（二极管/同步开关）$\to$ 被选通的输出支路
- **排错口令**："电感电流现在走哪条路？" —— 只要能回答这句，-600k 这种离谱值就很难困住你

#### 为什么能直接当排错工具
功率电路里最真实的不是电压，是**电流路径**：
- 电压离谱时，先问：电感电流到底流到哪儿去了？有没有通路？有没有被错误回流？
- MOSFET 接反/体二极管方向错，本质就是"以为有路"，实际是**把路堵死或引到反方向**

---

### 2. 系统结构与信号链路

#### 功率级（Power Stage）
```
Vin → 主开关(S1) → 单电感(L) → 分支节点 → [支路1/2/3] → 各自的 Cout + Rload → GND
```

#### 控制级（Control Stage）
对每一路 i∈{1,2,3}：
```
Vref_i ─┐
        ├─(误差 e_i=Vref_i−Vfb_i) → PI_i → Limiter(0..1) → 与 ramp 比较 → PWM_i
Vfb_i ──┘
```

#### 关键同步条件
- **ramp**：50kHz，幅值 0~1，三路共用 ✅
- **Limiter**：输出必须和 ramp 幅值匹配（0~1），否则比较器会"永远赢/永远输"

---

### 3. 控制链路解释（用类比讲清楚）

#### 链路结构
```
Vref → 误差计算(e = Vref − Vfb) → PI → Limiter(0..1) → Comparator vs ramp(0..1@50k) → PWM → Switch
```

#### PWM 的本质（不是"魔法波形"）
- **Comparator vs ramp** 本质上是在做"定时"：
  - ramp 是一个每周期从 0→1 的"计时尺"（50kHz）
  - Limiter 输出是你想要的"时间比例"（即占空比 $d$）
  - 比较器说：`当 ramp < d 时，PWM=1；否则 PWM=0`

- 所以只要 **ramp 与 d 同量纲**（0..1 对 0..1），PWM 的占空比就自然等于 $d$

#### 开环 vs 闭环（强力类比）
- **开环**：你固定给一个 $d$（比如 0.3），不管输出漂到哪。像"按经验每天吃同样剂量的药"。
- **闭环**：你测 Vfb，和 Vref 比，算出误差 $e$，再改 $d$。像"测血糖→算差→调胰岛素剂量"。

#### "PWM 在跳但开关不开"的原因
这正对应你踩的坑①：**MOSFET 浮地驱动导致 Vgs 不成立**
- Comparator 输出是"对地的电压/逻辑"，但 MOSFET 的开关判据是 $V_{gs} = V_{gate} - V_{source}$
- Source 漂了，$V_{gs}$ 可能很小，结果就是：**波形看起来在 PWM，器件实际上没真正导通**

### 三分钟排错剧本（3-step workflow）

#### Step 0（30s）：打开强制观测窗（同屏）
必看信号：
- `Vout1/2/3`, `Vfb1/2/3`, `e1/2/3`, `PIout`, `Limout`, `ramp`, `PWM1/2/3`, `IL`

**判据**：如果看不到 `d vs ramp` 与 `PWM` 的对应关系，就别改电路，先把观测补齐。

#### Step 1（60s）：看 Limout 是否卡死
1. 检查 `Limout` 是否在 0..1 **且不长期贴边**
2. 检查 `ramp` 是否 0..1 且频率正确（50kHz）
3. 检查 `Comparator` 极性：`ramp < d → PWM=1`（用固定 d=0.3 验证最直观）

**如果卡死**：先查误差极性、Limiter 范围、ramp 范围一致性

#### Step 2（60s）：做"参考点硬拉"验证
1. 把关键传感器负端直接拉到 VDC− 黑点节点
2. 观察输出是否立刻恢复

#### Step 3（30s）：确认 Vgs/开关真导通
1. 直接测 $V_{gs} = V_{gate} - V_{source}$ 是否足够
2. 或改用 Controlled/Ideal Switch 做对照

**如果 Vgs 太小**：根因 = 浮地驱动，改模型或做高边驱动参考

#### 隔离法 A：固定占空比替代 PI
- 断开 PI 输出 → 用 DC = 0.3/0.5 直接喂 Comparator
- 若输出能被拉起来 → 控制链大体没死，回头调 PI
- 若仍拉不起来 → 优先查开关模型/能量路径/参考点

#### 隔离法 B：单路跑通再叠加
- 只保留 V1（或只保留 V2）支路，把另外两路选通关死
- 能单路稳 → 功率级基本合理
- 单路都不稳 → 先把基础 Buck 跑稳，再谈三路耦合

## ❓ 常见问题与器件选型

### Q1：PSIM 里"红线/绿线"是什么意思？

**A**：两类网络分别承载不同信号类型：
- **Power 网络（红线）**：承载功率电流/电压的电气连接
- **Control/Logic 网络（绿线）**：承载控制信号（0/1、PWM、比较器输出）

你说的"从 on-off switch 出来就是红色，改不了"，说明：**你选的 on-off switch 是"功率器件"而不是"逻辑控制元件"**，它输出不是绿线，所以没法当 PWM 控制信号。

---

### Q2：On-Off Switch vs Bi-directional Switch 的区别？

| 器件 | 用途 | 控制方式 | 适用场景 |
|---|---|---|---|
| **On-Off Switch** | 开/关某条功率支路或测试隔离 | 不一定有控制端口（绿线） | 手动/固定隔离，**不适合 PWM 控制** |
| **Bi-directional Switch** | 功率级开关器件，可双向导通 | 需配套 Switch Controller 受控 | **支路选通/背靠背 MOSFET 等效** |

---

### Q3：你现在"支路选通/背靠背"场景应该用哪个？

**答案**：用 **Bi-directional Switch**（功率开关）+ **Switch Controller**（控制器）

**理由**：你需要"受 PWM 控制的功率路径选通"，而不是"手动开关"。

**PSIM 里的实际操作**：
1. 功率器件：**Power → Switches → Bi-directional Switch**（你截图里就在列表）
2. 控制器件：同一 Library Browser 左侧 → **Switch Controllers**（你截图里有文件夹名）
3. 用你的 PWM/逻辑信号（绿线）去驱动控制器，再由控制器去驱动 Bi-directional Switch

> 💡 提示："找不到 Controlled Switch (logic)" 是正常的 —— 你的库里就没这个名字。别硬搜字符串了，按上面两段式（Switch + Controller）走。

---

### Q4：为什么 MOSFET 浮地驱动这么要命？

**A**：MOSFET 导通取决于 $V_{gs} = V_{gate} - V_{source}$，不是"Gate 对地电压"。

- Comparator 输出是"对地的电压/逻辑"
- 但若 Source 在漂移点（比如在变压的节点上），$V_{gs}$ 可能很小
- 结果：**波形看起来在 PWM，器件实际没真正导通**

**快速检查**：直接测 $V_{gs}$ 的值；或改用 Controlled/Ideal Switch 完全避免这个问题。

### 模块 1：Error + PI + Limiter（每路一个）

## 📚 系统工程讲解

### 1. SITO 能量路径（核心本质）

#### 一句话模型
一个电感电流 $I_L$ 被时间复用，按时隙把能量"分发"给不同输出；**分发失败**通常不是 PI 算坏了，而是"路径/参考点/器件模型"让能量根本没按预期路走 [^2]

#### 三个动作模块

**① 充能（Charge）**：主开关导通，电感电流上升
- 直觉：电感像"能量背包"，ON 时往背包里塞能量
- 路径：$V_{in} \to$ 主开关 S1 $\to$ L（充能）$\to$ 被选通的输出支路

**② 分发（Allocate）**：电感放能，但"吐给谁"由支路选通决定
- SITO 的核心是"分发阀门"：同一股 $I_L$ 分时喂给 V1/V2/V3 [^2]
- 分发失败最常见不是控制坏了，而是：选通器件/回路/参考点问题 [^2]

**③ 续流（Freewheel）**：主开关关断后，电感电流必须有路走
- 路径：L（放能）$\to$ 续流路径（二极管/同步开关）$\to$ 被选通的输出支路
- **排错口令**："电感电流现在走哪条路？" —— 回答好这句，-600k 离谱值就很难困住你

---

### 2. 系统结构与信号链路

#### 功率级（Power Stage）
```
Vin → 主开关(S1) → 单电感(L) → 分支节点 → [支路1/2/3] → 各自的 Cout + Rload → GND
```

#### 控制级（Control Stage）
对每一路 i∈{1,2,3} [^4]：
```
Vref_i ─┐
        ├─(误差 e_i=Vref_i−Vfb_i) → PI_i → Limiter(0..1) → 与 ramp 比较 → PWM_i
Vfb_i ──┘
```

#### 关键同步条件
- **ramp**：50kHz，幅值 0~1，三路共用 ✅
- **Limiter**：输出必须和 ramp 幅值匹配（0~1），否则比较器会"永远赢/永远输" [^1]

---

### 3. 控制链路讲解（PWM / 闭环 / 开环）

#### PWM 的本质
**脉宽调制**：用"开关开多久"来控制平均能量 [^5]

类比：水龙头"开-关-开-关"，你控制"每次开着的时间比例"，就控制平均流量

- **Comparator vs ramp** 本质上是在做"定时"：
  - ramp 是一个每周期从 0→1 的"计时尺"
  - Limiter 输出是你想要的"时间比例"（即占空比 $d$）
  - 比较器说：`当 ramp < d 时，PWM=1；否则 PWM=0`

- 只要 **ramp 与 d 同量纲**（0..1 对 0..1），PWM 的占空比就自然等于 $d$

#### 开环 vs 闭环
- **开环**：固定给一个 $d$（比如 0.3），不管输出漂到哪。像"按经验每天吃同样剂量的药"
- **闭环**：测 Vfb，和 Vref 比，算出误差 $e$，再改 $d$。像"测血糖→算差→调胰岛素剂量"

#### 控制链路的脆弱点
你的链路：`e → PI → Limiter(0..1) → Comparator vs ramp(0..1@50k) → PWM → Switch` [^6]

| 模块 | 问题 | 优先级 |
|---|---|---|
| **PI** | 可能饱和、积分 wind-up | ⭐⭐ |
| **Limiter** | 量纲错（范围不是 0..1） | ⭐⭐⭐⭐⭐ |
| **Comparator** | 极性反 | ⭐⭐⭐⭐ |
| **MOSFET** | 浮地导致 Vgs 不成立 | ⭐⭐⭐⭐⭐ |

| **器件** | 开关模型 | MOSFET | Ideal/Controlled 优先 | 高边浮地需确保 Vgs [^7] |

## ⚠️ 致命坑表（现象 → 根因 → 验证 → 修复）

| 坑 | 典型现象 | 根因 | 最快验证（10秒） | 修复方案 |
|---|---|---|---|---|
| **参考点不一致** | 输出 -600k、测量不对、80≠100 | 你以为同地，实际不同节点 [^8] | 传感器负端拉到 VDC− 黑点 [^8] | 统一参考点 [^8] |
| **MOSFET 浮地** | PWM 在跳，支路不开 | Gate 对地，Source 漂移，Vgs 实际很小 [^8] | 测 Vgs=V(gate)-V(source) [^8] | 改 Ideal/Controlled [^8] |
| **Limiter 顶死** | VC1/2/3 全 0 或全 1 | 误差极性错/量纲不匹配 [^9] | 同屏看 e、PIout、Limout、ramp [^9] | 校验极性；确认 Lim=0..1 [^9] |
| **回流路径错** | 输出起不来、振荡、异常波形 | 体二极管方向错/回流不通 [^8] | 断开支路逐个验证 [^8] | 按论文核对方向 [^8] |

  │    │
  │    ├─→ A1: 同屏查看 Limout 是否卡 0/1？
  │    │       ├─ 是 → 修 "误差极性/量纲匹配"(e 方向, Lim=0..1, ramp=0..1)
  │    │       └─ 否 → A2

  │    └─→ A2: 统一参考点验证（传感器负端硬拉到 VDC−）

  │

       │

       │        ├─ 否 → 修测量点（END）

## 📋 三分钟排错剧本

### Step 0（30s）：打开强制观测窗（同屏）
必看信号 [^10]：
- `Vout1/2/3`, `Vfb1/2/3`, `e1/2/3`, `PIout`, `Limout`, `ramp`, `PWM1/2/3`, `IL`

**判据**：如果看不到 `d vs ramp` 与 `PWM` 的对应关系，就别改电路，先把观测补齐

### Step 1（60s）：看 Limout 是否卡死
1. 检查 `Limout` 是否在 0..1 **且不长期贴边** [^11]
2. 检查 `ramp` 是否 0..1 且频率正确（50kHz）
3. 检查 `Comparator` 极性：`ramp < d → PWM=1` [^11]

**如果卡死**：先查误差极性、Limiter 范围、ramp 范围一致性

### Step 2（60s）：做"参考点硬拉"验证
1. 把关键传感器负端直接拉到 VDC− 黑点节点 [^12]
2. 观察输出是否立刻恢复

**如果立刻正常**：根因 = 参考点不一致，统一所有地后 END

**如果仍异常**：进入 Step 3

### Step 3（30s）：确认 Vgs/开关真导通
1. 直接测 $V_{gs} = V_{gate} - V_{source}$ 是否足够 [^12]
2. 或改用 Controlled/Ideal Switch 做对照

**如果 Vgs 太小**：根因 = 浮地驱动，改模型或做高边驱动参考

### 模块 1：Error + PI + Limiter（每路一个）
- **输入**：$V_{ref,i}$, $V_{fb,i}$
- **输出**：$d_i$（0..1）
- **判据**：稳态时 $d_i$ 在 0..1 中间某个值，不应长期卡死边界 [^16]

### 模块 2：Ramp + Comparator → PWM
- **输入**：$d_i$（0..1）、ramp（0..1）

- **判据**：$d_i$ 越大，PWM 高电平占比越大

- **输入**：$PWM_i$
- **输出**：支路导通状态
- **注意**：若用 MOSFET，高边/浮地必须确保 Vgs；否则用 Controlled/Ideal Switch [^17]

## ❓ 常见问题与 PSIM 器件选型

### Q1：PSIM 里"红线/绿线"是什么意思？

在 PSIM 中：
- **Power 网络（红线）**：承载功率电流/电压的电气连接
- **Control/Logic 网络（绿线）**：承载控制信号（0/1、PWM、比较器输出）

你说的"从 on-off switch 出来就是红色，改不了"，说明：**你选的 on-off switch 是"功率器件"而不是"逻辑控制元件"**，它输出不是绿线，所以没法当 PWM 控制信号

### Q2：On-Off Switch vs Bi-directional Switch 的区别？

| 器件 | 用途 | 控制方式 | 适用场景 |
|---|---|---|---|
| **On-Off Switch** | 开/关某条功率支路或测试隔离 | 不一定有控制端口（绿线） | 手动/固定隔离，**不适合 PWM 控制** |
| **Bi-directional Switch** | 功率级开关器件，可双向导通 | 需配套 Switch Controller 受控 | **支路选通/背靠背 MOSFET 等效** |

### Q3：你现在应该用哪个？

**答案**：用 **Bi-directional Switch**（功率开关）+ **Switch Controller**（控制器）

**理由**：你需要"受 PWM 控制的功率路径选通"，而不是"手动开关"

**PSIM 操作步骤**：
1. 功率器件：**Power → Switches → Bi-directional Switch**
2. 控制器件：同一 Library Browser 左侧 → **Switch Controllers**
3. 用你的 PWM/逻辑信号（绿线）去驱动控制器，再由控制器去驱动 Bi-directional Switch

## 🎓 核心概念速记

### PWM 的本质
**脉宽调制** = 用"开关开多久"来控制平均能量 [^5]

### 闭环 vs 开环
- **开环**：固定 $D$，系统不管输出变没变（蒙眼开车）
- **闭环**：实时测 $V_{fb}$ 和目标 $V_{ref}$ 比较，自动调 $D$（看着车道线纠偏）

### SITO 三输出共电感的关键
**不是 PI 怎么算**，而是**"电感能量在某时刻主要喂给哪一路"由支路选通器件决定** [^18]
