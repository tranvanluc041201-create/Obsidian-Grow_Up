## 6）越界 / 溢出（按“对象边界”理解，最清晰）

### 6.1 概念（别混）

* **越界读（OOB Read）**：读了不属于对象边界的内存
* **越界写（OOB Write）**：写了不属于对象边界的内存（更危险）
* **缓冲区溢出（Buffer Overflow）**：写太多导致越界写（最常见）
* //理解:不小心-不注意缓冲的大小而导致,前面两种属于新手错误
* **整数溢出（Integer Overflow）**：数值算错 -> 常导致分配大小/长度计算错 -> 间接越界

### 6.2 经典案例：少留 '\0'（越界写）

```c
char buf[4]; // 只能放 3 个字符 + '\0'
buf[0]='A'; buf[1]='B'; buf[2]='C'; buf[3]='D';
buf[4]='\0'; // 🚨 越界写：buf[4] 不存在
```

```text
[Stack Snapshot - 越界写覆盖隔壁]
Address      Value    属于谁
0x1000       'A'      buf[0]
0x1001       'B'      buf[1]
0x1002       'C'      buf[2]
0x1003       'D'      buf[3]    <- buf 的边界到此为止
0x1004       '\0'     other_x   <- 🚨 覆盖别人的内存
```

### 6.3 后果（从常见到严重）

1. 数据悄悄错（最难排）
2. 崩溃（segmentation fault）
3. 复杂场景形成安全风险（了解即可）
