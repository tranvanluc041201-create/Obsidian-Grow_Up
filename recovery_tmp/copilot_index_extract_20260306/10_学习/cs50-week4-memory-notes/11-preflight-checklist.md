## 11）强制自检清单（写代码前扫一遍，极大降低出错率）

### 指针/内存

* [ ] 解引用前确认指针合法（必要时判 NULL）
* [ ] malloc 大小：`n * sizeof(type)`（别漏 sizeof）
* [ ] 每个 malloc 都有对应 free（包括中途 return 的分支）
* [ ] 数组/字符串边界：预留 '\0'；循环用 `<` 而不是 `<=`

### 文件

* [ ] `fopen` 后判 `fp == NULL`
* [ ] 想保留历史：用 "a"；想重建文件：用 "w"；只读：用 "r"
* [ ] 写完 `fclose`（确保刷新落盘）
* [ ] `fscanf` 检查返回值；`%s` 必加宽度限制

### 4）质量门槛（每次提交/自测必过）

**4.1 编译必须 0 warning**
- `make clean && make`

**4.2 运行场景必须覆盖（你自己写成脚本更好）**
- add 两次（验证 "a"）
- list（验证 "r"）
- del 一条（验证内存操作 + 保存）
- 故意把文件路径改错（验证 fopen 判 NULL）
- 故意制造脏行（验证解析稳健）

**4.3 Valgrind 必须干净（至少做到这个）**
- 无 Invalid read/write
- 无 definitely lost
