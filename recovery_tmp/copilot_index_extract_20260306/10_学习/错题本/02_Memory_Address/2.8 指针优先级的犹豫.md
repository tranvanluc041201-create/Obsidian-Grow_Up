### [2.8] 指针优先级的犹豫
* **标签**：#Pointers #Precedence
* **错误**：混淆 `*` 解引用与 `++` 的优先级和结合方向。
* **原题**：
```c
int arr[] = {10, 20, 30};
int *p = arr;
int y = (*p)++;
```
* **正确结果**：`y=10, arr[0]=11, p` 仍指向 `arr[0]`。
* **根因**：`(*p)++` 先解引用，再对内存对象自增；返回的是旧值。
* **对比**：`*p++` 等价于 `*(p++)`，指针先移动，再解引用旧地址。
* **修正**：用括号明确意图，避免阅读歧义。
