## 🎯 Problem（问题）

printf输出时类型不匹配导致数据错误：

```c
// score定义为int型，但输出格式要求float型
printf("%5.1f", score);  // ❌ 类型不匹配
```

**问题**：`int`读取4字节，`double`读取8字节，剩余4字节会读取垃圾数据。

---

## 💥 发生了什么

- **预期**：正确输出分数值
- **实际**：输出错误或垃圾值
- **原因**：
  - `int`占4字节
  - `%f`期望`double`（8字节）
  - 从栈上多读了4字节垃圾数据

---

## 🤔 当时我是怎么想的

> "格式字符串只是提示怎么显示，实际类型会自动转换..."> "int和float差不多，应该能混用..."> 
> **盲区**：不知道printf是**可变参数函数**，类型不匹配时不会自动转换。

---

## 🔍 Analysis（分析）

### 表面原因
> printf格式字符串与实际参数类型不匹配

### 深层原理

**printf的工作方式**：
```c
printf("%5.1f", score);
// %f 告诉printf：从参数栈上读8字节（double）
// 但score是int，只占4字节
// 结果：读了4字节score + 4字节垃圾数据
```

**可变参数函数的局限**：
- 编译器不知道参数应该是什么类型
- 完全依赖格式字符串
- **类型不匹配 = 未定义行为**

**常见类型匹配**：
| 格式 | 期望类型 | 常见错误 |
|------|---------|---------|
| `%d` | int | 传float |
| `%f` | double | 传int |
| `%c` | char/int | 传char* |
| `%s` | char* | 传char |
| `%p` | void* | 传其他类型 |

### 模式识别
> **printf格式必须与参数类型严格匹配**。

## ✅ Refix（修正）

```c
int score = 85;

// ❌ 错误：类型不匹配
printf("%5.1f", score);

// ✅ 正确1：使用%d输出整数
printf("%d", score);

// ✅ 正确2：先转换为float
printf("%5.1f", (float)score);

// ✅ 正确3：存储时就定义为float
float score_f = 85.0;
printf("%5.1f", score_f);
```

## 💡 Insight（高阶认知）

### 同类变式
- **scanf的类型匹配**
  ```c
  int x;
  scanf("%f", &x);  // ❌ 错误！会向int地址写入float数据
  // 正确：scanf("%d", &x);
  ```

### 工程启示
> **printf/scanf安全使用**：> 1. 格式字符串与实际参数严格匹配> 2. 使用 `-Wformat` 编译器警告检查> 3. 现代C考虑使用类型安全的替代方案（如C++的iostream）

|--------|---------|------|
| **%d** | int | 不是float |
| **%f** | double | 不是int |
