## 🎯 Problem（问题）

if-else配对理解错误导致逻辑错误：

```c
#include <stdio.h>
int main() {
    int x, y = 2;
    scanf("%d", &x);
    if (x >= 2)
        if (x >= 6)
            y = 168;
        else
            y = 8;
    printf("%d", y);
}
```

**问题**：输入x=1时，输出y=2（不是8）

---

## 💥 发生了什么

- **错误理解**：`else` 和第一个 `if` 配对
- **实际**：`else` 和**最近的未配对**的 `if` 配对
- **结果**：`x=1` 时，第一个if条件不满足，内部整个if-else都不执行

**正确缩进揭示真相**：
```c
if (x >= 2)
    if (x >= 6)
        y = 168;
    else        // ← 这个else属于第二个if！
        y = 8;
```

> **盲区**：不知道C语言**忽略缩进**，else总是和最近的if配对。

## 🔍 Analysis（分析）

### 表面原因
> 误解if-else配对规则，被缩进误导

### 深层原理

**C语言if-else配对规则**：
- `else` 与**最近的、未配对的** `if` 配对
- 缩进只是给人看的，编译器**完全忽略**

**对比**：
```c
// 缩进误导
if (a)
    if (b)
        x = 1;
else        // 看起来和第一个if配对
    x = 2;

// 实际逻辑（编译器看到的）
if (a)
    if (b)
        x = 1;
    else        // 和第二个if配对！
        x = 2;
```

### 模式识别
> **看到嵌套if-else时，用花括号明确边界**。

## ✅ Refix（修正）

```c
// ✅ 方法1：用花括号明确代码块
if (x >= 2) {
    if (x >= 6)
        y = 168;
    else
        y = 8;
}

// ✅ 方法2：如果else要和第一个if配对
if (x >= 2) {
    if (x >= 6)
        y = 168;
} else {
    y = 8;
}

// ✅ 方法3：始终使用花括号（推荐）
if (x >= 2) {
    if (x >= 6) {
        y = 168;
    } else {
        y = 8;
    }
}
```

## 💡 Insight（高阶认知）

### 同类变式
- **更复杂的嵌套**
  ```c
  if (a) if (b) x = 1; else x = 2;  // ❌ 难以阅读
  
  // ✅ 始终使用花括号
  if (a) {
      if (b) {
          x = 1;
      } else {
          x = 2;
      }
  }
  ```

### 工程启示
> **编码规范（Google C++ Style）**：
> - **始终**使用花括号，即使只有一行
> - 避免嵌套if-else超过3层
> - 复杂逻辑用switch或提前return简化
