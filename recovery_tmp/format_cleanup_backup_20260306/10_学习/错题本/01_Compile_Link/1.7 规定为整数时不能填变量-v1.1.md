## 🎯 Problem（问题）

变量定义和科学计数法使用错误：

```c
int n = 6, score6;
1e-n       // ❌ 错误：1e后面只能跟整数，不能是变量
scoren     // ❌ 错误：变量名不能嵌套变量
```

---

## 💥 发生了什么

- **错误1**：`1e-n` 想表达科学计数法，但`n`是变量
- **错误2**：`scoren` 想表达`score[n]`，但写成了变量名

---

## 🤔 当时我是怎么想的

> "1e-n应该是1乘以10的-n次方..."> "scoren应该是score的第n个元素..."> 
> **盲区**：不知道C语言语法规则限制了这些写法。

---

## 🔍 Analysis（分析）

### 表面原因
> 混淆了数学表达和C语言语法

### 深层原理

**科学计数法规则**：
```c
// ✅ 正确：1e后面必须是整数常量
1e6     // 1 × 10^6 = 1000000
1.5e-3  // 1.5 × 10^-3 = 0.0015

// ❌ 错误：不能用变量
1e-n    // 语法错误！
```

**变量命名规则**：
```c
int score6;   // ✅ 变量名score6
int score[6]; // ✅ 数组score的第6个元素
scoren;       // ❌ 变量名scoren（如果未定义就是错误）
score[n];     // ✅ 数组score的第n个元素（n是变量）
```

### 模式识别
> **C语言语法是严格的，数学直觉不能直接套用**。

---

## ✅ Refix（修正）

```c
// ❌ 原始错误代码
int n = 6, score6;
1e-n
scoren

// ✅ 修正1：科学计数法只能是常量
double val = 1e-6;  // 常量
// 如果需要变量指数，用pow函数
#include <math.h>
double val = pow(10, -n);  // 10的-n次方

// ✅ 修正2：数组访问用方括号
int score[10];
score[n] = 100;  // 访问第n个元素
```

## 💡 Insight（高阶认知）

### 工程启示
> **C语言与数学的区别**：
> - 科学计数法只能是编译期常量
> - 变量索引必须用方括号
> - 变量名是标识符，不能"动态生成"
