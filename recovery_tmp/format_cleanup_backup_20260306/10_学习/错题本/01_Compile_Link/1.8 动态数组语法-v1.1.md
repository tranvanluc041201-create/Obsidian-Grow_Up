## 🎯 Problem（问题）

试图用变量创建"动态数组"：

```c
int n = 6;
int a[n];  // ❌ 错误理解：以为n改变，数组大小也会改变
```

或：
```c
int n = 0;
int a[n];  // ❌ n=0，数组大小为0，后续访问越界
```

---

## 💥 发生了什么

**错误理解**：
- 以为`a[n]`会随着`n`的变化动态调整大小
- 以为可以在运行时改变数组大小

**实际**：
- C语言数组大小在**定义时确定**
- 之后**不能改变**
- VLA（变长数组）虽然可以用变量定义，但大小也是**一次性确定**的

---

## 🤔 当时我是怎么想的

> "既然n是变量，那a[n]应该就是动态数组..."> "n改变后，数组应该自动跟着变..."> 
> **盲区**：不知道C语言数组大小**编译期确定**（或VLA定义时确定），之后固定不变。

---

## 🔍 Analysis（分析）

### 表面原因
> 误以为C语言数组可以动态调整大小

### 深层原理

**C语言数组的本质**：
```c
int a[10];  // 编译器分配 10 * sizeof(int) = 40字节
// 这40字节是固定的，不能扩展或收缩
```

**VLA（C99变长数组）的局限**：
```c
int n = 10;
int a[n];  // VLA，大小在定义时确定为10
n = 20;    // 数组大小仍然是10！不会变
```

**真正的动态数组**：
```c
// 用malloc动态分配
int *a = malloc(10 * sizeof(int));  // 10个int
// 需要扩展时：
a = realloc(a, 20 * sizeof(int));    // 扩展到20个int
```

### 模式识别
> **C语言数组大小固定，需要动态大小用malloc/realloc**。

## ✅ Refix（修正）

```c
// ❌ 错误理解
int n = 6;
int a[n];
n = 10;  // 数组大小仍然是6！

// ✅ 方法1：预分配最大可能大小
#define MAX_SIZE 100
int a[MAX_SIZE];
int n = 6;  // 实际使用6个

// ✅ 方法2：使用malloc（真正的动态数组）
int *a = malloc(n * sizeof(int));
// 需要扩展时：
a = realloc(a, new_n * sizeof(int));
```

## 💡 Insight（高阶认知）

### 同类变式
- **动态数组的最佳实践**
  ```c
  typedef struct {
      int *data;
      int size;
      int capacity;
  } Vector;
  
  void vector_push(Vector *v, int val) {
      if (v->size >= v->capacity) {
          v->capacity *= 2;
          v->data = realloc(v->data, v->capacity * sizeof(int));
      }
      v->data[v->size++] = val;
  }
  ```

### 工程启示
> **数组选择指南**：
> - 大小固定且已知：用普通数组
> - 大小运行时确定但不变：用VLA（C99）或malloc
> - 大小需要动态变化：用malloc/realloc或动态数组结构
> - 现代C++：用std::vector
