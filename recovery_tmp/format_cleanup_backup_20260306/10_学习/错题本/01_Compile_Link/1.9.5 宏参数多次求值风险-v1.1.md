## 🎯 Problem（问题）

宏定义中参数被多次展开导致副作用：

```c
#define MONITOR_BUF(ptr, len) ((ptr) + (len) > (ptr) ? (ptr) : NULL)

char data[10];
char *p_data = data;
int offset = 5;

char *p_final = MONITOR_BUF(p_data++, offset++);
```

**预期**：`offset=6`，指针后移一次  
**实际**：`offset=6`（这个对了），但如果`ptr`有副作用...

---

## 💥 发生了什么

**关键问题**：宏中`ptr`出现了3次！
```c
// MONITOR_BUF(p_data++, offset++) 展开：
((p_data++) + (offset++) > (p_data++) ? (p_data++) : NULL)
```

如果写成 `MONITOR_BUF(p_data++, 5)`：
- `p_data++` 会执行**3次**！
- 指针被错误地移动了3个位置

**三目运算符中的求值**：
- 条件为真时：`ptr`被求值2次（条件中2次 + 结果1次）
- 注意：C标准未规定`>`左右两边的求值顺序

---

## 🤔 当时我是怎么想的

> "宏参数只会在调用时求值一次..."> "三目运算符只会执行一个分支..."> 
> **盲区**：不知道宏是**纯文本替换**，参数出现几次就被替换几次。

---

## 🔍 Analysis（分析）

### 表面原因
> 宏参数在宏体中出现多次，导致多次求值

### 深层原理

**宏展开的本质**：
```c
#define MONITOR_BUF(ptr, len) ((ptr) + (len) > (ptr) ? (ptr) : NULL)

MONITOR_BUF(p_data++, 5)
// 展开：
((p_data++) + (5) > (p_data++) ? (p_data++) : NULL)
// p_data++ 出现了3次！执行3次自增！
```

**对比函数调用**：
```c
char *monitor_buf(char *ptr, int len) {
    return (ptr + len > ptr) ? ptr : NULL;
}
monitor_buf(p_data++, 5);  // ✅ p_data++ 只执行一次！
```

### 模式识别
> **宏参数如果含有++/--/函数调用，绝不要传入宏**。

---

## ✅ Refix（修正）

```c
// ❌ 危险用法
char *p_final = MONITOR_BUF(p_data++, offset++);

// ✅ 方法1：先求值到临时变量
char *p_temp = p_data++;
int len_temp = offset++;
char *p_final = MONITOR_BUF(p_temp, len_temp);

// ✅ 方法2：改用inline函数（推荐）
static inline char *monitor_buf(char *ptr, int len) {
    return (ptr + len > ptr) ? ptr : NULL;
}
char *p_final = monitor_buf(p_data++, offset++);  // 安全！

// ✅ 方法3：GCC扩展语句表达式
#define MONITOR_BUF(ptr, len) ({ \
    typeof(ptr) _ptr = (ptr); \
    typeof(len) _len = (len); \
    (_ptr + _len > _ptr) ? _ptr : NULL; \
})
```

## 💡 Insight（高阶认知）

### 同类变式
- **更隐蔽的例子**
  ```c
  #define MIN(a, b) ((a) < (b) ? (a) : (b))
  int x = 5;
  int m = MIN(x++, 10);  // x++ 执行1次还是2次？取决于比较结果！
  ```

### 横向链接
- **关联错题**：[[1.9.2] 宏定义副作用幽灵]

### 工程启示
> **绝对禁止**：
> - 将`++`/`--`作为宏参数
> - 将函数调用作为宏参数
> - 将可能有副作用的表达式作为宏参数> 
> **硬件后果**：> 在嵌入式系统中，这种错误可能导致：
> - 指针越界访问（HardFault）
> - 寄存器写错地址
> - 系统崩溃
