## 🎯 Problem（问题）

指针自增运算优先级混淆：

```c
int b = *p++;  // ❓ 意图是数值+1，但...
```

**实际结果**：先取值，然后指针后移（不是数值+1！）

---

## 💥 发生了什么

- **意图**：让指针指向的数值加1
- **实际**：取当前值，然后指针指向下一个位置
- **等价写法**：`*(p++)`

---

## 🤔 当时我是怎么想的

> "`*p`是取值，`++`是自增，合起来就是让值自增..."> 
> **盲区**：不知道后缀`++`优先级高于`*`，且`p++`是**先返回旧值，再自增指针**。

---

## 🔍 Analysis（分析）

### 表面原因
> 混淆了`*p++`和`(*p)++`的含义

### 深层原理

**优先级与结合性**：
| 表达式 | 等价写法 | 结果 |
|--------|---------|------|
| `*p++` | `*(p++)` | 取`*p`的值，然后`p`后移 |
| `(*p)++` | — | 取`*p`的值，然后`*p`自增 |
| `*++p` | `*(++p)` | `p`先前移，然后取`*p` |
| `++*p` | `++(*p)` | `*p`先自增，然后取值 |

**关键区别**：
- `p++`：表达式值是旧地址，p自身增加
- `++p`：表达式值是新地址，p自身增加

### 模式识别
> **指针+自增的4种写法含义完全不同**，必须加括号明确意图。

---

## ✅ Refix（修正）

```c
// ❌ 错误意图：想让值+1
int b = *p++;  // 实际：取值，指针后移

// ✅ 正确写法1：明确让值+1
int b = (*p)++;  // 取*p的值，然后*p自增

// ✅ 正确写法2：拆成两行（推荐！清晰）
int b = *p;  // 取值
(*p)++;      // 值自增

// ✅ 正确写法3：前置自增
++*p;  // 或 (*p)++，但++*p更简洁
```

## 💡 Insight（高阶认知）

### 同类变式
- **更复杂的场景**
  ```c
  int arr[] = {10, 20, 30};
  int *p = arr;
  
  printf("%d\n", *p++);   // 输出10，p指向arr[1]
  printf("%d\n", *p);     // 输出20
  ```

## 🎯 关键记忆点

| 表达式 | 口诀 |
|--------|------|
| `*p++` | 先取值，指针后移 |
| `(*p)++` | 取值并自增，指针不动 |
| `*++p` | 指针先移，再取值 |
| `++*p` | 值先自增，再取值 |
