## 🎯 Problem（问题）

指针与自增运算符的优先级混淆：

```c
int arr[] = {10, 20, 30};
int *p = arr;
int y = (*p)++;  // ❓ y是多少？arr[0]是多少？p指向哪？
```

---

## 💥 发生了什么

- **y = 10**（旧值）
- **arr[0] = 11**（被自增了）
- **p 仍指向 arr[0]**（指针没动）

对比另一个常见错误：
```c
int y = *p++;  // ❌ 完全不同！
// y = 10, arr[0] = 10, p指向arr[1]
```

---

## 🤔 当时我是怎么想的

> "`*p++` 应该是先解引用，再自增指针吧..."
> 
> **盲区**：不知道 `++` 和 `*` 的优先级，以及 `p++` 返回旧值的特性。

---

## 🔍 Analysis（分析）

### 表面原因
> 混淆了 `(*p)++` 和 `*p++` 的运算顺序

### 深层原理

**优先级与结合性**：
| 表达式 | 等价写法 | 执行顺序 | 结果 |
|--------|---------|---------|------|
| `*p++` | `*(p++)` | 先p++(返回旧地址)，再解引用 | 取arr[0]，p移向arr[1] |
| `(*p)++` | — | 先解引用，再对值自增 | 取arr[0]并自增，p不动 |
| `*++p` | `*(++p)` | 先++p(p指向arr[1])，再解引用 | 取arr[1]，p移向arr[1] |
| `++*p` | `++(*p)` | 先解引用，再对值前置自增 | arr[0]变11，表达式值11 |

**关键区别**：
- `p++`：**后自增**，表达式值是旧地址，p自身指向新位置
- `++p`：**前自增**，表达式值是新地址，p自身指向新位置

### 模式识别
> **指针+自增的4种写法**：`*p++`、`(*p)++`、`*++p`、`++*p`，每种含义都不同！

## ✅ Refix（修正）

```c
// ✅ 明确意图：取值，然后指针后移
int val = *p;   // 取值
p++;            // 指针后移（清晰，推荐！）

// ✅ 明确意图：取值并自增，指针不动
(*p)++;         // 等价于 arr[0]++

// ✅ 明确意图：指针先移到下一个，再取值
++p;            // 或 p++
int val = *p;

// ❌ 不推荐：一行写多个副作用
int y = *p++ + *++p;  // 未定义行为！不要这样写
```

**推荐原则**：一行只写一个副作用（自增/自减/赋值），代码清晰优先。

## 💡 Insight（高阶认知）

### 同类变式
- **数组版本的迷惑**
  ```c
  int arr[3] = {10, 20, 30};
  int *p = arr;
  
  printf("%d\n", *p++);   // 输出10，p指向arr[1]
  printf("%d\n", *p);     // 输出20
  printf("%d\n", (*p)++); // 输出20，arr[1]变成21
  printf("%d\n", arr[1]); // 输出21
  ```

- **函数参数中的陷阱**
  ```c
  func(*p++);  // 先取*p作为参数，再p++
  func((*p)++); // 先取*p，然后*p自增
  ```

## 🎯 关键记忆点

| 写法 | 记忆口诀 |
|------|---------|
| `*p++` | 先取值，指针后移 |
| `(*p)++` | 取值并自增，指针不动 |
| `*++p` | 指针先移，再取值 |
| `++*p` | 值先自增，再取值 |

**黄金法则**：一行只写一个副作用！
