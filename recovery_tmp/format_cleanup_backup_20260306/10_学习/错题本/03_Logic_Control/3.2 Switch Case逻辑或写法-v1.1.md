## 🎯 Problem（问题）

Switch Case中误用逻辑或：

```c
switch (c) {
    case 'a' || 'A':  // ❌ 逻辑错误！
        printf("Vowel\n");
        break;
}
```

**错误直觉**：匹配'a'或'A'  
**实际结果**：等价于`case 1:`（永远匹配）

---

## 💥 发生了什么

**逻辑或运算**：
- `'a' || 'A'` = 非零 || 非零 = 真 = **1**
- 等价于 `case 1:`

**正确写法**：
```c
case 'a':  // 穿透到下一个case
case 'A':
    // 处理代码
```

---

## 🤔 当时我是怎么想的

> "既然是或的关系，用||应该可以..."> "case后面可以写表达式吧..."> (可以用但是不能用变量比较大小)
> **盲区**：不知道case后面必须是**常量表达式**，且逻辑或的结果是0或1。

---

## 🔍 Analysis（分析）

### 表面原因
> 误用逻辑或`||`，应该用case穿透

### 深层原理

**case标签的要求**：
- 必须是**常量表达式**（编译期确定）
- 不能包含变量
- 不能包含函数调用

**合法的case**：
```c
case 5:           // ✅ 常量
case 'A':         // ✅ 字符常量（就是整数）
case 1 + 2:       // ✅ 常量表达式（编译期计算为3）
case (70 > 60):   // ✅ 常量表达式（结果为1）

// case score > 60:  // ❌ 含变量
case func():      // ❌ 函数调用
case x + 3:       // ❌ 含变量
```

**Case穿透（Fall-through）**：
- 多个case共享同一段代码时，不加break
- 这是C语言的标准用法

### 模式识别
> **Switch匹配多个值**：用case穿透，不是逻辑或。

---

## ✅ Refix（修正）

```c
// ✅ 正确写法：case穿透
switch (c) {
    case 'a':
    case 'e':
    case 'i':
    case 'o':
    case 'u':
    case 'A':
    case 'E':
    case 'I':
    case 'O':
    case 'U':
        printf("Vowel\n");
        break;
    default:
        printf("Consonant\n");
}

// ✅ 或者用if-else（更灵活）
if (c == 'a' || c == 'A' || c == 'e' || c == 'E' || ...)
    printf("Vowel\n");
```

## 💡 Insight（高阶认知）

### 同类变式
- **重复case值**
  ```c
  int x = 65;
  switch (x) {
      case 'A': printf("1"); break;  // ❌ 错误：'A'就是65
      case 65:  printf("2"); break;  // 与上一行重复！
  }
  ```

- **字符的本质**
  ```c
  'A' == 65  // ✅ 真，字符就是整数
  ```

### 横向链接
- **关联错题**：[[3.5] 逻辑短路的冷酷规则]（同样是逻辑运算符问题）

### 工程启示
> **Switch vs If-else选择**：>
>  - 离散整数/字符值：用switch，编译器可能优化为跳转表> 
>    - 范围判断或复杂条件：用if-else
> - 多个值共享逻辑：用case穿透> 
> **编译器优化**：> 当case密集时（如0~255），编译器可能生成跳转表（O(1)）> 比if-else链（O(n)）更高效
