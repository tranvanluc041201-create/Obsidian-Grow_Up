## 🎯 Problem（问题）

同一表达式中多次修改同一变量：

```c
int j = 5;
int q = (++j) + (++j) + (++j);  // ❌ 未定义行为！
```

---

## 💥 发生了什么

**未定义行为（Undefined Behavior, UB）**：
- 不同编译器结果可能不同
- 同一编译器不同优化级别结果也可能不同

**可能的结果**：
- GCC: 可能算出 24 (6+7+8 或 8+8+8)
- 其他编译器: 结果可能不同

---

## 🤔 当时我是怎么想的

> "从左到右计算，应该是6+7+8=21..."> "或者先全部自增，再相加..."> 
> **盲区**：不知道C标准没有规定这种表达式的求值顺序。

---

## 🔍 Analysis（分析）

### 未定义行为的原因

C标准规定：**在同一个序列点之间，不能多次修改同一变量，也不能同时读取和修改同一变量**。

```c
(++j) + (++j) + (++j)
// j被修改了3次，但只有一个序列点（表达式结束时）
// 未定义行为！
```

### 序列点（Sequence Point）
- 逻辑与 `&&` 的左侧
- 逻辑或 `||` 的左侧
- 条件运算符 `?:` 的第一个操作数
- 完整表达式结束（语句结束）
- 函数调用前的所有参数求值完成后

---

## ✅ Refix（修正）

```c
// ❌ 危险代码
q = (++j) + (++j) + (++j);

// ✅ 正确：拆成多个语句
++j;
++j;
++j;
q = j + j + j;  // 或者 q = 3 * j;

// ✅ 正确：明确意图
j += 3;
q = 3 * j;
```

---

## 💡 Insight（高阶认知）

### 常见的UB
| 代码 | 问题 |
|------|------|
| `i++ + i++` | 两次修改i |
| `a[i] = i++` | 同时读写i |
| `f(++i, ++i)` | 参数求值顺序未定义 |

### 工程启示
> **避免UB的黄金法则**：
> - 一行只写一个副作用（++/--/赋值）
> - 不要依赖求值顺序
> - 开启编译器警告 `-Wall -Wsequence-point`
