## 🎯 Problem（问题）

关系运算符优先级陷阱导致死循环：

```c
int x = 3, y;
do {
    y = x--;
    if (!y) { printf("x"); continue; }
    printf("#");
} while (1 <= x <= 2);  // ❌ 永远为真！
```

**预期**：循环在x不满足条件时结束  
**实际**：死循环

---

## 💥 发生了什么

**错误理解**：以为 `1 <= x <= 2` 会限制x在1到2之间

**实际执行**：
```c
// C语言从左向右结合：
(1 <= x) <= 2

// 1 <= x 的结果是 0 (假) 或 1 (真)
// 0 和 1 都 <= 2

// 结论：条件永远为真！
```

---

## 🤔 当时我是怎么想的

> "1 <= x <= 2 就是数学里的 1≤x≤2..."> "这样写应该能限制x的范围..."> 
> **盲区**：不知道C语言关系运算符是从左向右结合，不是数学的区间表达。

---

## 🔍 Analysis（分析）

### 关系运算符结合性

```c
// ❌ C语言不支持数学区间写法
1 <= x <= 2  // 等价于 (1 <= x) <= 2，永远为真

// ✅ 正确写法：用逻辑与
x >= 1 && x <= 2

// ✅ 正确写法：用逻辑与（另一种）
1 <= x && x <= 2
```

### 模式识别
> **C语言关系运算符是二元的**，不支持链式比较。

---

## ✅ Refix（修正）

```c
// ❌ 错误
while (1 <= x <= 2)  // 死循环！

// ✅ 正确
while (x >= 1 && x <= 2)

// ✅ 正确（另一种）
while (1 <= x && x <= 2)
```

---

## 💡 Insight（高阶认知）

### 对比其他语言
| 语言 | 链式比较 | 示例 |
|------|---------|------|
| **C/C++** | ❌ 不支持 | `1 <= x && x <= 2` |
| **Python** | ✅ 支持 | `1 <= x <= 2` |
| **数学** | ✅ 支持 | `1 ≤ x ≤ 2` |
