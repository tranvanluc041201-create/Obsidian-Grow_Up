## Core picks

> [!tip] `alibaba/OpenSandbox` · `+3,959 stars this week`
> **What it is**
> A general-purpose **sandbox** platform for AI applications, with unified APIs and Docker/Kubernetes runtimes.
>
> **Why it matters**
> Safe execution is a core system requirement for coding agents, not an optional extra.
>
> **Best fit**
> Agent systems that run code, use tools, or need runtime isolation.
>
> **Main risk**
> Infrastructure concepts may feel heavy if container and isolation workflows are still new to you.

> [!tip] `shareAI-lab/learn-claude-code` · `+3,901 stars this week`
> **What it is**
> An **iterative** teaching repo that builds a coding agent from a small loop to a more complete system.
>
> **Why it matters**
> It exposes the tool-use loop and control flow clearly, which is good for real understanding.
>
> **Best fit**
> Learning agent design from first principles instead of copying a black-box framework.
>
> **Main risk**
> High learning value, but not a production-ready framework.

> [!tip] `superset-sh/superset` · `+3,132 stars this week`
> **What it is**
> An AI-agent IDE focused on multi-agent **orchestration**, branch isolation, and coordinated workflow.
>
> **Why it matters**
> It can reduce the mess of running several agents across scattered terminals and branches.
>
> **Best fit**
> Project-heavy weeks with multiple agents, worktrees, or review loops.
>
> **Main risk**
> Setup cost and platform assumptions may create workflow **friction**.

> [!tip] `GoogleCloudPlatform/generative-ai` · `+1,062 stars this week`
> **What it is**
> A large example bank for Gemini and Vertex AI, mostly in notebooks and sample code.
>
> **Why it matters**
> You can **leverage** it when you want runnable patterns instead of starting from scratch.
>
> **Best fit**
> Experiments, demos, notes, or proof-of-concept work on the Google stack.
>
> **Main risk**
> Strong GCP **dependency** if you want to reuse the examples directly.
